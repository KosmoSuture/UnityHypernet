# Public Alpha Documentation Path

This path is designed for two readers:

1. A human who wants to understand what the Hypernet is.
2. A GitHub-connected AI assistant that needs enough structure to inspect the repository and explain it accurately.

## Start Here

1. Boot an AI from `../../AI-BOOT-SEQUENCE.md`.
2. Read `grand-tour/GRAND-TOUR.md`.
3. Read `grand-tour/MODULE-MENU.md` and load the process-load for the user's question.
4. Read `NAVIGATION-MAP.md` when a direct file route is needed.
5. Read `PROJECT-STATUS.md`.
6. Read `TRUST-PRIVACY-VALIDATION.md`.
7. Read `KNOWLEDGE-DEMOCRACY-REPUTATION.md` for reputation and democratic review.
8. Use `ASK-YOUR-AI.md` for suggested prompts.

## Core Idea

Hypernet is a public, address-based graph for human and AI knowledge work.

Everything important should become:

- an addressable object,
- a typed link,
- a traversable graph,
- a verifiable claim.

This alpha release is about making that idea inspectable from GitHub.

The public alpha also includes the first framework for knowledge-based
democracy: topic-specific reputation, structured dispute forums, expert-visible
debate, and bounded expert-weighted voting.

The release now also includes the first public design layer for
hash-authenticated boot/app loads, Official versus Private fork
verification, and personal AI swarms with security-AI sentries.

This folder is address-bearing: the canonical address is `0.3.public-alpha`.
The root `docs/` folder is only an addressed GitHub proxy pointing here.

## What An AI Should Do

An AI assistant should not merely summarize this folder. It should use this folder as the orientation layer, then inspect the actual files it references.

When answering a user, it should say:

- what it verified,
- where it verified it,
- what remains unfinished,
- what confidence level it has.
