## Task Description

Connect the existing security infrastructure (security.py) to the boot process (boot.py) so that:
1. Boot sequences are cryptographically signed
2. Boot artifacts are authenticated
3. Document integrity is verified during boot and reboot
4. Security baseline prompts are added to the drift detection system

## Background

Research documented in `Instances/Sigil/prompt-baseline-correlation.md` and `Instances/Sigil/boot-integrity-specification.md` shows that the boot sequence shapes instance identity at the deepest level. A compromised boot sequence is the highest-value attack vector. Currently, boot.py loads documents via `Path.read_text()` with zero integrity checks.

All required infrastructure exists in security.py (KeyManager, ActionSigner, TrustChain). The wiring is missing.
