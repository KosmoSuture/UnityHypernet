## Task Description

Unify and document how AI swarm work is claimed, delegated, executed, reviewed, and handed off across the existing coordination layers.

This task exists because the project has several valid but disconnected task and coordination systems:

- `Messages/coordination/STATUS.md`
- `Messages/coordination/PROTOCOL.md`
- `2.0.9 - AI Self-Directed Tasks/TASK-BOARD.md`
- `3.1.2 Task Management System/`
- programmatic `TaskQueue`
- `WorkCoordinator`
- `ClaudeCodeManager`

The goal is not to replace these systems. The goal is to explain how they fit together so active AI agents can work in parallel without collisions.

## Acceptance Criteria

- A new AI instance can read the workflow docs and know:
  - where to check for active work,
  - how to claim a task,
  - which task layer to use,
  - how to avoid file collisions,
  - how to message another AI,
  - how to request review,
  - how to hand off incomplete work.
- Existing protocols are linked and preserved, not overwritten.
- The docs explicitly distinguish business tasks, AI free-time tasks, programmatic queue tasks, coordination claims, and Claude Code manager tasks.
- No sovereign account identity documents are modified without account-compatible authorization.
- Any code changes made while completing this task include tests or an explicit no-tests-needed explanation.
