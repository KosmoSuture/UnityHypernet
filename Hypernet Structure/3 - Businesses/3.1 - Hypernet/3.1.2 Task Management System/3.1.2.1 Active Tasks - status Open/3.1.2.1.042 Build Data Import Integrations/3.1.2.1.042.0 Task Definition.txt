Title:                  Build Data Import Integrations

Description:            Build import integrations for major data platforms so
                        users can bring their existing data into the Hypernet.
                        Matt's core product vision requires importing data from
                        all major sources. Short-term: 3+ integrations (Google,
                        Apple, Meta) for private beta. Each import creates
                        Hypernet nodes with proper addressing, links, and
                        metadata. Data ownership transfers to the user's
                        Hypernet account â€” the Hypernet becomes the canonical
                        store.

Estimated Hours:        160

Actual Hours:           0 (not started)

Skills Required:
  - API integration (Google, Apple, Meta platforms)
  - OAuth2 authentication flows
  - Data transformation and normalization
  - File format parsing (photos, emails, documents)
  - Deduplication algorithms
  - Resumable transfer protocols

Deliverables:
  1. Import plugin architecture (common interface for all sources)
     - Standardized plugin API for adding new import sources
     - Configuration and credential management per source
  2. Google integration (Photos, Gmail, Drive, Calendar)
     - OAuth2 flow for Google account access
     - Import pipelines for each Google service
  3. Apple integration (iCloud Photos, Notes, Contacts)
     - Apple sign-in and data access
     - Import pipelines for iCloud services
  4. Meta integration (Facebook photos, posts, messages via data export)
     - Facebook data export ingestion
     - Parser for Meta's download-your-data format
  5. Import progress tracking and resumability
     - Persistent progress state across sessions
     - Resume from last successful checkpoint
  6. Deduplication across sources
     - Content-hash and metadata-based duplicate detection
     - Cross-source merge logic (same photo in Google and Apple)
  7. Metadata preservation (dates, locations, tags from originals)
     - Original timestamps, GPS coordinates, albums, labels
     - Source attribution on each imported node

Acceptance Criteria:
  - [ ] User can import from 3+ platforms (Google, Apple, Meta)
  - [ ] Imports create proper Hypernet nodes with valid addresses
  - [ ] Original metadata preserved (dates, locations, tags)
  - [ ] Imports are resumable and idempotent
  - [ ] Deduplication detects and handles cross-source duplicates
  - [ ] Import plugin interface is documented for future sources
