# 1.0.0 - Comprehensive Person Structure Definition

## Purpose

This document defines EVERY type of data a person might want to store in their Hypernet account. This is the complete specification for the 1.* node structure.

**Hypernet Address:** `1.0.0`
**Applies To:** All living people (1.1+)
**Last Updated:** February 10, 2026

---

## Philosophy: Your Complete Digital Life

Hypernet is designed to store **everything** about a person's digital and physical life:
- Every email, photo, video, document
- Every password, credential, secret
- Every financial transaction, account, investment
- Every medical record, prescription, appointment
- Every relationship, connection, interaction
- Every task, goal, achievement
- Every thought, note, idea
- Every purchase, subscription, asset
- Everything you might need, reference, or cherish

**Privacy First:** You control what's stored, who sees it, and how it's used.

## Privacy & Security Settings

### Per-Category Privacy Controls

Each category can have different privacy settings:

**Privacy Levels:**
1. **Private:** Only you
2. **Family:** You + designated family members
3. **Friends:** You + designated friends
4. **Professional:** You + professional network
5. **Public:** Anyone (use sparingly)
6. **AI Access:** Which AI companies can access
7. **Legacy:** What becomes public after death

### Security Controls

- Two-factor authentication
- Biometric authentication
- End-to-end encryption (for sensitive categories)
- Access logs and audit trails
- Granular permissions per AI company
- Data retention settings
- Right to deletion
- Data export

## Database Implementation

### Hypernet Address Format

```
1.X.Y.Z.NNNNN
│ │ │ │ └─ Instance number (00001-99999)
│ │ │ └─ Sub-category
│ │ └─ Category
│ └─ Person ID
└─ Living People
```

**Example:**
```
1.1.6.1.00142 = Matt's Photo #142
  └ 1 = Living People
    └ 1 = Matt
      └ 6 = Personal Data
        └ 1 = Media Objects
          └ 00142 = Instance
```

### Object Types

Each category corresponds to object types defined in 0.5 - Universal Objects:
- Photo → 0.5.2.1
- Email → 0.5.3.1
- Transaction → 0.5.6.2
- HealthRecord → 0.5.6.4
- Task → 0.5.5.1
- etc.

## Usage Examples

### Example 1: Family Photo

```
Object: Photo from family dinner
HA: 1.1.6.1.00142

Properties:
- title: "Mom's 67th Birthday Dinner"
- taken_at: 2026-02-09T18:30:00Z
- location: Giuseppe's Italian Restaurant
- camera: iPhone 15 Pro
- people: [1.1 Matt, 1.2 Sarah, 1.3-1.7 Kids, Mom]

Links:
- TAKEN_BY: 1.1 (Matt)
- DEPICTS: 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 5.X.Y.Z (Mom, when in People of History)
- OCCURRED_DURING: Event "Family Dinner"
- LOCATED_AT: Giuseppe's Restaurant
- EMOTIONAL_CONTEXT: "Joyful, warm, celebratory"
```

### Example 2: Medical Record

```
Object: Doctor's Appointment
HA: 1.1.6.8.00089

Properties:
- record_type: appointment
- date: 2026-01-15
- provider: Dr. Smith
- facility: Main Street Medical
- reason: Annual checkup
- diagnosis: Healthy
- prescriptions: None
- follow_up: One year

Privacy: PRIVATE (only 1.1 can access)
AI Access: NONE (medical data not shared)
```

### Example 3: Financial Transaction

```
Object: Purchase at Giuseppe's
HA: 1.1.6.7.00234

Properties:
- amount: $87.43
- merchant: Giuseppe's Italian Restaurant
- category: Dining
- payment_method: Credit Card ending 4532
- date: 2026-02-09

Links:
- OCCURRED_DURING: Event "Family Dinner"
- SPLIT_WITH: 1.2 Sarah (Venmo $43.72)
- RECEIPT: Document object
- BUDGET_CATEGORY: "Special Occasions"
```
