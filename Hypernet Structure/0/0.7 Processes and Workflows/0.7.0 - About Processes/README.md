# 0.7.0 - About Processes and Workflows

Processes and workflows define how things happen in the Hypernet. While objects (0.5) define what exists and links (0.6) define how things relate, workflows define how things change, transform, and move through the system.

## Workflow Categories

| Address | Category | Description |
|---------|----------|-------------|
| 0.7.1 | Governance Workflows | Elections, proposals, policy changes |
| 0.7.2 | Contribution Workflows | Onboarding, work logging, credit, payment |
| 0.7.3 | Review Workflows | Peer review, verification, trust scoring |
| 0.7.4 | Incident Response | Security, failures, disputes, emergencies |
| 0.7.5 | AI Workflows | Boot, reboot, task execution, code review |
| 0.7.6 | Data Workflows | Import, export, archive, migration |

## Design Principles

1. **Transparency**: All steps are visible and logged
2. **Automation**: Routine steps are automated where possible
3. **Human oversight**: Critical decisions require human judgment
4. **Fail-safe**: Defined error states and recovery procedures

## Relationship to Existing Files

The flat files (`0.7.0 Processes and Workflows Overview.md` through `0.7.4 Incident Response Workflows.md`) contain the original specifications. The folder structure provides deeper decomposition into individual workflow definitions.
