# 0.7.2 Contribution Workflows

## Overview

Contribution workflows manage how people participate in building the Hypernet, track their work, and receive their share of the three-way profit split.

## Contributor Onboarding Workflow

### Purpose
Welcome and integrate new contributors.

```yaml
onboarding_workflow:
  workflow_id: "WF-CONTRIB-001"
  name: "Contributor Onboarding"

  steps:
    - step_id: "O1"
      name: "Registration"
      actions:
        - "Create Hypernet account"
        - "Basic identity verification"
        - "Accept contributor agreement"
      outputs:
        - "Mandala ID created"
        - "Initial reputation: 0.2"

    - step_id: "O2"
      name: "Profile Setup"
      actions:
        - "Complete skills inventory"
        - "Set availability"
        - "Specify interests"
        - "Choose contribution areas"

    - step_id: "O3"
      name: "Orientation"
      actions:
        - "Complete orientation modules"
        - "Review governance structure"
        - "Understand profit sharing model"
        - "Learn tools and processes"
      required_modules:
        - "Hypernet Mission and Values"
        - "How Contribution Tracking Works"
        - "Collaboration Tools"
        - "Governance Participation"

    - step_id: "O4"
      name: "Mentor Assignment"
      optional: true
      actions:
        - "Match with experienced contributor"
        - "Initial guidance session"
        - "30-day check-in scheduled"

    - step_id: "O5"
      name: "First Contribution"
      actions:
        - "Browse available tasks"
        - "Claim or be assigned first task"
        - "Complete with mentor support"
        - "Log contribution"

    - step_id: "O6"
      name: "Verification"
      actions:
        - "First contribution reviewed"
        - "Feedback provided"
        - "Full contributor status granted"
      outputs:
        - "Reputation boost: +0.05"
        - "Access to more task types"
```

## Work Logging Workflow

### Purpose
Track contributions for profit sharing.

```yaml
work_logging_workflow:
  workflow_id: "WF-CONTRIB-002"
  name: "Contribution Logging"

  trigger: "Contributor completes work"

  steps:
    - step_id: "L1"
      name: "Log Entry"
      actor: "contributor"
      actions:
        - "Record work session"
      required_fields:
        date: "date of work"
        hours: "time spent"
        project: "[Task/Project reference]"
        description: "what was done"
        work_type: "code|content|design|management|support"
        paid: "boolean"

    - step_id: "L2"
      name: "Evidence Attachment"
      optional_but_recommended: true
      actions:
        - "Attach deliverables"
        - "Link commits/documents"
        - "Provide verification links"

    - step_id: "L3"
      name: "Submission"
      actions:
        - "Entry submitted for verification"
        - "Automatic checks run"
      automatic_checks:
        - "Hours within reasonable range"
        - "No duplicate entries"
        - "Task exists and is active"

    - step_id: "L4"
      name: "Verification"
      actor: "verifier"
      verification_options:
        peer_verification:
          description: "Another contributor verifies"
          requirements: "verifier reputation >= 0.6"
        manager_verification:
          description: "Project manager verifies"
          requirements: "manager role on project"
        auto_verification:
          description: "System verifies via evidence"
          requirements: "sufficient linked evidence"

    - step_id: "L5"
      name: "Credit Calculation"
      automatic: true
      calculation:
        paid_work: "hours × 1 = credit_hours"
        unpaid_work: "hours × 10 = credit_hours"
      outputs:
        - "credit_hours added to contributor total"
        - "entry marked as verified"

    - step_id: "L6"
      name: "Confirmation"
      actions:
        - "Contributor notified"
        - "Entry visible in contribution log"
        - "Running totals updated"
```

## Credit Distribution Workflow

### Purpose
Calculate and distribute profit shares.

```yaml
distribution_workflow:
  workflow_id: "WF-CONTRIB-003"
  name: "Profit Share Distribution"

  trigger: "monthly|quarterly as configured"

  steps:
    - step_id: "D1"
      name: "Period Close"
      actions:
        - "Lock contribution entries for period"
        - "No new entries for closed period"
        - "Calculate total credit hours"

    - step_id: "D2"
      name: "Pool Calculation"
      actions:
        - "Calculate net revenue for period"
        - "Apply three-way split"
        - "Determine contributor pool amount"
      calculation: |
        net_revenue = gross_revenue - operating_expenses
        contributor_pool = net_revenue × 0.333

    - step_id: "D3"
      name: "Share Calculation"
      actions:
        - "Sum all verified credit hours"
        - "Calculate per-hour value"
        - "Compute individual shares"
      calculation: |
        total_credits = sum(all_contributor_credits)
        credit_value = contributor_pool / total_credits
        individual_share = contributor_credits × credit_value

    - step_id: "D4"
      name: "Vesting Application"
      actions:
        - "Apply vesting schedule"
        - "25% immediately available"
        - "75% vests over 48 months"

    - step_id: "D5"
      name: "Review Period"
      duration: "7 days"
      actions:
        - "Preliminary calculations published"
        - "Contributors can review"
        - "Disputes can be raised"

    - step_id: "D6"
      name: "Dispute Resolution"
      conditional: "if disputes raised"
      actions:
        - "Review disputed entries"
        - "Adjustments made if warranted"
        - "Final calculations published"

    - step_id: "D7"
      name: "Payment Processing"
      actions:
        - "Initiate payments"
        - "Multiple payment methods supported"
        - "Tax documentation generated"
      payment_methods:
        - "Bank transfer"
        - "Cryptocurrency"
        - "PayPal/similar"
        - "Contribution to project (in-kind)"

    - step_id: "D8"
      name: "Confirmation"
      actions:
        - "Payment confirmations sent"
        - "Period marked complete"
        - "Records archived"
```

## Skill Verification Workflow

### Purpose
Verify contributor skills for task matching.

```yaml
skill_verification_workflow:
  workflow_id: "WF-CONTRIB-004"
  name: "Skill Verification"

  steps:
    - step_id: "S1"
      name: "Skill Claim"
      actor: "contributor"
      actions:
        - "Select skill to verify"
        - "Indicate proficiency level"
        - "Provide evidence"

    - step_id: "S2"
      name: "Evidence Review"
      verification_methods:
        portfolio:
          description: "Review work samples"
          verifier: "expert in skill area"
        assessment:
          description: "Complete skill assessment"
          verifier: "automated or expert"
        endorsement:
          description: "Endorsements from verified peers"
          requirements: "3+ endorsements from verified practitioners"
        certification:
          description: "External certification"
          verifier: "document verification"

    - step_id: "S3"
      name: "Verification Decision"
      outcomes:
        verified:
          actions: ["Skill badge added", "Task eligibility updated"]
        needs_improvement:
          actions: ["Feedback provided", "Resources suggested"]
        not_verified:
          actions: ["Can retry after improvement"]
```
