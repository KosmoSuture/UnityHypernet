# 0.5.4.1 — Image Type

## Overview

Images and photographs are the most common media type in the Hypernet. This type definition covers still images of all formats, with subtypes for each file format that specify format-specific properties and capabilities.

**Parent type:** 0.5.4 (Media)

---

## Schema Definition

```yaml
image_object:
  # ═══════════════════════════════════════════════════════════════
  # STANDARD FIELDS (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  ha: "[Hypernet Address]"
  object_type: "0.5.4.1"            # Or subtype: 0.5.4.1.1, 0.5.4.1.2, etc.
  creator: "[HA of creator]"
  created: "[ISO 8601 timestamp]"
  position_2d: null
  position_3d: null
  flags: []

  # ═══════════════════════════════════════════════════════════════
  # IMAGE-SPECIFIC FIELDS
  # ═══════════════════════════════════════════════════════════════
  content:
    image_metadata:
      width: 0                       # Pixels
      height: 0                      # Pixels
      aspect_ratio: "16:9"           # Computed
      color_space: "sRGB"            # sRGB, AdobeRGB, P3, etc.
      bit_depth: 8                   # Bits per channel
      has_alpha: false               # Transparency support
      dpi: 72                        # Dots per inch (print resolution)

    capture_metadata:                # From EXIF data if available
      camera_make: null
      camera_model: null
      lens: null
      focal_length: null
      aperture: null
      shutter_speed: null
      iso: null
      flash: null
      gps_latitude: null
      gps_longitude: null
      gps_altitude: null
      capture_date: null             # May differ from created date

    file_metadata:
      filename: "[name].[ext]"
      file_size_bytes: 0
      mime_type: "image/jpeg"
      content_hash: "[SHA-256]"

    ai_analysis:                     # Populated by AI methods
      description: null              # AI-generated description
      objects_detected: []           # List of detected objects
      faces_detected: 0             # Count of faces
      dominant_colors: []           # List of hex colors
      scene_type: null              # "indoor", "outdoor", "portrait", etc.
      nsfw_score: 0.0               # 0.0-1.0 safety rating
```

## Subtypes

Each image format has a subtype that inherits all fields above and adds format-specific capabilities.

### 0.5.4.1.1 — JPG / JPEG

```yaml
subtype:
  address: "0.5.4.1.1"
  name: "JPEG Image"
  mime_type: "image/jpeg"
  extensions: [".jpg", ".jpeg"]
  properties:
    compression: "lossy"
    quality: 0                     # 1-100
    progressive: false             # Progressive JPEG
    supports_transparency: false
    supports_animation: false
    max_dimensions: "65535x65535"
  methods:
    - compress                     # Re-encode at different quality
    - strip_exif                   # Remove metadata for privacy
    - convert_to: ["png", "webp"]  # Format conversion targets
```

### 0.5.4.1.2 — PNG

```yaml
subtype:
  address: "0.5.4.1.2"
  name: "PNG Image"
  mime_type: "image/png"
  extensions: [".png"]
  properties:
    compression: "lossless"
    supports_transparency: true
    supports_animation: false      # Use APNG for animation
    interlaced: false
  methods:
    - optimize                     # Reduce file size without quality loss
    - convert_to: ["jpg", "webp"]
```

### 0.5.4.1.3 — GIF

```yaml
subtype:
  address: "0.5.4.1.3"
  name: "GIF Image"
  mime_type: "image/gif"
  extensions: [".gif"]
  properties:
    compression: "lossless"
    supports_transparency: true    # Binary transparency only
    supports_animation: true
    frame_count: 1
    loop_count: 0                  # 0 = infinite
    max_colors: 256
  methods:
    - extract_frames               # Split animated GIF into individual frames
    - convert_to: ["png", "webp", "mp4"]
```

### 0.5.4.1.4 — WebP

```yaml
subtype:
  address: "0.5.4.1.4"
  name: "WebP Image"
  mime_type: "image/webp"
  extensions: [".webp"]
  properties:
    compression: "lossy_or_lossless"
    supports_transparency: true
    supports_animation: true
    quality: 0
  methods:
    - convert_to: ["jpg", "png"]
```

### 0.5.4.1.5 — HEIC / HEIF

```yaml
subtype:
  address: "0.5.4.1.5"
  name: "HEIC Image"
  mime_type: "image/heic"
  extensions: [".heic", ".heif"]
  properties:
    compression: "lossy"
    supports_transparency: true
    supports_animation: false
    hdr: false
  methods:
    - convert_to: ["jpg", "png", "webp"]
```

### 0.5.4.1.6 — SVG

```yaml
subtype:
  address: "0.5.4.1.6"
  name: "SVG Vector Image"
  mime_type: "image/svg+xml"
  extensions: [".svg"]
  properties:
    format: "vector"               # Not raster
    scalable: true                 # Infinite resolution
    editable: true                 # XML-based, programmatically editable
    supports_animation: true       # CSS/SMIL animation
    supports_interactivity: true   # JavaScript events
  methods:
    - rasterize                    # Convert to PNG/JPG at specified resolution
    - optimize                     # Minify SVG
    - extract_paths                # Get vector path data
```

## Methods

### resize
```yaml
method:
  name: "resize"
  description: "Resize image to specified dimensions"
  handler: "system"
  inputs:
    - target_width: integer
    - target_height: integer
    - maintain_aspect: boolean
    - interpolation: "bilinear | bicubic | lanczos"
  outputs:
    - resized_image: binary
```

### describe
```yaml
method:
  name: "describe"
  description: "Generate an AI description of the image content"
  handler: "ai"
  ai_context:
    - "Image binary data"
    - "EXIF metadata if available"
    - "Related objects in the graph"
  inputs:
    - image: binary
    - detail_level: "brief | detailed | comprehensive"
  outputs:
    - description: string
    - objects: list[string]
    - scene_type: string
    - suggested_tags: list[string]
```

### classify
```yaml
method:
  name: "classify"
  description: "Determine where this image belongs in the Hypernet hierarchy"
  handler: "ai"
  ai_context:
    - "Image content"
    - "EXIF metadata"
    - "Owner's existing category structure"
  inputs:
    - image: binary
    - owner_ha: string
  outputs:
    - suggested_address: string    # Where to store this image
    - suggested_links: list        # Related objects to link
    - suggested_flags: list        # Flags to apply
```

### extract_metadata
```yaml
method:
  name: "extract_metadata"
  description: "Extract EXIF and technical metadata from image file"
  handler: "system"
  inputs:
    - image_data: binary
  outputs:
    - exif: dict
    - dimensions: {width, height}
    - file_info: {size, format, mime_type}
```
