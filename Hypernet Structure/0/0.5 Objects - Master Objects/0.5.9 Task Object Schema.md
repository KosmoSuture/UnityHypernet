# 0.5.9 Task Object Schema

## Overview

Task Objects represent work to be done—action items, projects, milestones, and any trackable unit of work. They are central to collaboration and project management within the Hypernet.

## Schema Definition

```yaml
task_object:
  # ═══════════════════════════════════════════════════════════════
  # BASE OBJECT (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "[org].[system].[status].[instance]"  # e.g., 3.1.2.1.001
    object_id: "[UUID]"
    object_type: "task"
    subtype: "action_item|project|milestone|epic|story|bug"
    version: "1.0.0"

  # ═══════════════════════════════════════════════════════════════
  # TASK-SPECIFIC CONTENT
  # ═══════════════════════════════════════════════════════════════
  content:
    # ─────────────────────────────────────────────────────────────
    # TASK IDENTITY
    # ─────────────────────────────────────────────────────────────
    task_identity:
      title: "Task Title"
      description: "Detailed description of what needs to be done"

      task_id: "TASK-001"  # Human-readable identifier
      external_ids:
        - system: "github"
          id: "issue-123"
        - system: "jira"
          id: "PROJ-456"

    # ─────────────────────────────────────────────────────────────
    # STATUS AND PRIORITY
    # ─────────────────────────────────────────────────────────────
    status:
      current: "open|assigned|in_progress|review|blocked|complete|cancelled"

      history:
        - status: "open"
          timestamp: "2025-01-30T00:00:00Z"
          by: "[Person reference]"
        - status: "assigned"
          timestamp: "2025-01-30T10:00:00Z"
          by: "[Person reference]"

      blocked:
        is_blocked: false
        blocker: "[Task/Issue reference]"
        reason: "Waiting for API specification"

    priority:
      level: "critical|high|medium|low"
      score: 85  # Numerical priority for sorting

    # ─────────────────────────────────────────────────────────────
    # ASSIGNMENT
    # ─────────────────────────────────────────────────────────────
    assignment:
      assigned_to:
        - person: "[Person Object reference]"
          role: "primary"
          assigned_date: "2025-01-30T10:00:00Z"
          assigned_by: "[Person reference]"

      team: "[Organization/Team reference]"

      skills_required:
        - skill: "Python"
          level: "intermediate"
        - skill: "API Design"
          level: "advanced"

    # ─────────────────────────────────────────────────────────────
    # TIMING
    # ─────────────────────────────────────────────────────────────
    timing:
      created: "2025-01-30T00:00:00Z"
      due_date: "2025-02-15T23:59:59Z"

      estimates:
        original_hours: 40
        current_hours: 45
        confidence: 0.7

      tracking:
        hours_logged: 12
        percentage_complete: 30

        time_entries:
          - date: "2025-01-31"
            hours: 4
            person: "[Person reference]"
            description: "Initial research"
          - date: "2025-02-01"
            hours: 8
            person: "[Person reference]"
            description: "Implementation started"

      milestones:
        - name: "Design Complete"
          target_date: "2025-02-01"
          status: "complete"
          completed_date: "2025-01-31"
        - name: "Implementation Complete"
          target_date: "2025-02-10"
          status: "in_progress"

    # ─────────────────────────────────────────────────────────────
    # DELIVERABLES
    # ─────────────────────────────────────────────────────────────
    deliverables:
      - deliverable_id: "D1"
        description: "API specification document"
        type: "document"
        status: "complete"
        artifact: "[Document reference]"

      - deliverable_id: "D2"
        description: "Working implementation"
        type: "code"
        status: "in_progress"
        artifact: null  # Not yet delivered

    acceptance_criteria:
      - criterion: "API responds in under 100ms"
        testable: true
        status: "not_tested"

      - criterion: "All unit tests pass"
        testable: true
        status: "passing"

      - criterion: "Documentation is complete"
        testable: true
        status: "not_tested"

    # ─────────────────────────────────────────────────────────────
    # DEPENDENCIES
    # ─────────────────────────────────────────────────────────────
    dependencies:
      upstream:  # Tasks that must complete before this one
        - task: "[Task reference]"
          type: "blocks"
          status: "complete"

      downstream:  # Tasks waiting on this one
        - task: "[Task reference]"
          type: "blocked_by"

      related:  # Related but not dependent
        - task: "[Task reference]"
          relationship: "similar"

    # ─────────────────────────────────────────────────────────────
    # HIERARCHY
    # ─────────────────────────────────────────────────────────────
    hierarchy:
      parent: "[Parent Task/Project reference]"

      children:
        - task: "[Subtask reference]"
          order: 1
        - task: "[Subtask reference]"
          order: 2

      epic: "[Epic reference]"
      milestone: "[Milestone reference]"

    # ─────────────────────────────────────────────────────────────
    # COLLABORATION
    # ─────────────────────────────────────────────────────────────
    collaboration:
      comments:
        - comment_id: "C1"
          author: "[Person reference]"
          timestamp: "2025-01-31T14:00:00Z"
          content: "Started working on this. Will update by EOD."
          reactions:
            - type: "thumbs_up"
              by: "[Person reference]"

      activity_log:
        - timestamp: "2025-01-30T10:00:00Z"
          action: "assigned"
          by: "[Person reference]"
          details: "Assigned to [Person]"

      watchers:
        - "[Person reference]"

      mentions:
        - person: "[Person reference]"
          in: "comment_C1"

    # ─────────────────────────────────────────────────────────────
    # CONTRIBUTION TRACKING (for Hypernet profit sharing)
    # ─────────────────────────────────────────────────────────────
    contribution:
      contribution_tracking_enabled: true

      contributions:
        - contributor: "[Person reference]"
          hours: 12
          type: "unpaid"  # Gets 10x credit
          credit_hours: 120
          verified_by: "[Person reference]"

      total_credit_hours: 120

    # ─────────────────────────────────────────────────────────────
    # BUDGET (if applicable)
    # ─────────────────────────────────────────────────────────────
    budget:
      allocated: {amount: 5000, currency: "USD"}
      spent: {amount: 1200, currency: "USD"}
      remaining: {amount: 3800, currency: "USD"}

      expenses:
        - description: "Cloud hosting for development"
          amount: {amount: 200, currency: "USD"}
          date: "2025-01-31"
          receipt: "[Document reference]"

  # ═══════════════════════════════════════════════════════════════
  # LINKS
  # ═══════════════════════════════════════════════════════════════
  links:
    # Assignment
    - link_type: "assigned_to"
      target: "[Person address]"
      metadata: {role: "primary"}

    # Parent project
    - link_type: "part_of"
      target: "[Project address]"

    # Dependencies
    - link_type: "depends_on"
      target: "[Task address]"

    - link_type: "blocks"
      target: "[Task address]"

    # Related documents
    - link_type: "references"
      target: "[Document address]"

    # Deliverables
    - link_type: "produced"
      target: "[Object address]"
```

## Task Types

### Action Item
Single, discrete unit of work.

### Project
Collection of related tasks toward a larger goal.

### Milestone
Significant checkpoint or deliverable.

### Epic
Large body of work spanning multiple projects.

### Story
User-focused feature description (agile).

### Bug
Defect to be fixed.

## Integration with Contribution Tracking

Tasks integrate with Hypernet's three-way profit split:
- All work hours can be logged
- Unpaid hours receive 10× credit multiplier
- Credits determine share of contributor fund
