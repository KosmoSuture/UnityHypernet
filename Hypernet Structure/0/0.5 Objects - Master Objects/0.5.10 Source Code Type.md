# 0.5.10 — Source Code Type

## Overview

Source code is a first-class object type in the Hypernet. Code files are not just text — they have structure (classes, functions, imports), behavior (they can be executed, tested, linted), and relationships (imports, dependencies, version history). This type definition specifies how the Hypernet understands and interacts with code.

**Parent type:** 0.5.3 (Document)

Source control, API access, class properties and methods — all defined here so that AI workers and human developers interact with code through a standard interface.

## Schema Definition

```yaml
source_code:
  # ═══════════════════════════════════════════════════════════════
  # STANDARD FIELDS (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  ha: "[Hypernet Address]"
  object_type: "0.5.10"            # Or subtype: 0.5.10.1, 0.5.10.2, etc.
  creator: "[HA of creator]"
  created: "[ISO 8601 timestamp]"
  position_2d: null
  position_3d: null
  flags: []

  # ═══════════════════════════════════════════════════════════════
  # CODE-SPECIFIC FIELDS
  # ═══════════════════════════════════════════════════════════════
  content:
    code_metadata:
      language: "python"             # Language identifier
      language_version: "3.12"       # Target version
      encoding: "utf-8"
      line_count: 0
      content_hash: "[SHA-256]"

    file_metadata:
      filename: "[name].[ext]"
      file_size_bytes: 0
      mime_type: "text/x-python"
      filesystem_path: "[relative path]"

    structure:                       # Parsed from AST
      imports: []                    # List of {module, names, from}
      classes: []                    # List of {name, bases, methods, properties, line}
      functions: []                  # List of {name, params, return_type, line}
      constants: []                  # List of {name, value, line}
      docstrings: []                 # Module and function docstrings

    dependencies:
      internal: []                   # HAs of other code files this imports
      external: []                   # Package names from pip/npm/etc.
      version_constraints: {}        # {package: ">=1.0.0"}

    quality:                         # Populated by automated analysis
      test_coverage: null            # Percentage, if known
      lint_issues: 0
      complexity_score: null         # Cyclomatic complexity
      last_test_run: null            # ISO 8601
      last_test_result: null         # "pass" | "fail"

    source_control:
      repository: null               # URL or HA of the repository
      branch: null
      last_commit_hash: null
      last_commit_message: null
      last_commit_date: null
      contributors: []               # List of HAs
```

## Methods

### parse_ast
```yaml
method:
  name: "parse_ast"
  description: "Parse source code into abstract syntax tree and extract structure"
  handler: "system"
  inputs:
    - source: string
    - language: string
  outputs:
    - classes: list
    - functions: list
    - imports: list
    - constants: list
    - errors: list                   # Parse errors if any
```

### run_tests
```yaml
method:
  name: "run_tests"
  description: "Execute the test suite associated with this code"
  handler: "system"
  inputs:
    - test_file: string              # Path to test file
    - test_runner: string            # "pytest", "unittest", "jest", etc.
  outputs:
    - passed: integer
    - failed: integer
    - errors: list
    - coverage: float                # If coverage tool available
```

### review
```yaml
method:
  name: "review"
  description: "AI code review — analyze for bugs, style, security, performance"
  handler: "ai"
  ai_context:
    - "Full source code"
    - "Language type definition and conventions"
    - "Related code files (imports)"
    - "Test results if available"
    - "Project coding standards"
  inputs:
    - source: string
    - review_focus: list             # ["bugs", "security", "style", "performance"]
  outputs:
    - issues: list                   # {severity, line, description, suggestion}
    - overall_quality: string        # "good", "needs-work", "critical-issues"
    - suggestions: list
```

### refactor
```yaml
method:
  name: "refactor"
  description: "AI-assisted refactoring — restructure code while preserving behavior"
  handler: "ai"
  ai_context:
    - "Full source code"
    - "Test suite"
    - "Dependent files"
  inputs:
    - source: string
    - goal: string                   # "simplify", "extract_function", "rename", etc.
  outputs:
    - refactored_source: string
    - changes_description: string
    - files_affected: list
```

### explain
```yaml
method:
  name: "explain"
  description: "AI explains what this code does"
  handler: "ai"
  ai_context:
    - "Full source code"
    - "Module/class docstrings"
  inputs:
    - source: string
    - audience: string               # "beginner", "intermediate", "expert"
  outputs:
    - explanation: string
    - key_concepts: list
```

## API Access

Code objects in the Hypernet expose their public API surface. When another system needs to interact with a Python module, the type definition tells it:

```yaml
api_surface:
  classes:
    - name: "ClassName"
      address: "[HA of this code file].ClassName"
      properties:
        - {name: "prop_name", type: "str", access: "read"}
      methods:
        - {name: "method_name", params: ["arg1: str", "arg2: int"], returns: "bool"}
  functions:
    - name: "function_name"
      address: "[HA].function_name"
      params: ["arg1: str"]
      returns: "dict"
  constants:
    - name: "CONSTANT_NAME"
      value: "value"
```

This enables AI workers to discover and use code programmatically without reading the full source.
