# 0.5.2 Organization Object Schema

## Overview

Organization Objects represent collective entities—businesses, governments, non-profits, communities, and any other group of people working together. They provide structure for collaboration, resource management, and collective decision-making.

## Schema Definition

```yaml
organization_object:
  # ═══════════════════════════════════════════════════════════════
  # BASE OBJECT (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "[2|3].[category].[instance]"  # 2 for non-commercial, 3 for business
    object_id: "[UUID]"
    object_type: "organization"
    subtype: "business|government|nonprofit|community|informal"
    version: "1.0.0"

  # ═══════════════════════════════════════════════════════════════
  # ORGANIZATION-SPECIFIC CONTENT
  # ═══════════════════════════════════════════════════════════════
  content:
    # ─────────────────────────────────────────────────────────────
    # IDENTITY
    # ─────────────────────────────────────────────────────────────
    identity:
      legal_name: "Full Legal Organization Name"
      display_name: "Common Name"
      aliases: ["Other names", "Abbreviations"]
      logo: "[Media Object reference]"
      description: "Organization mission and description"

      legal_structure:
        type: "corporation|llc|nonprofit|cooperative|dao|informal"
        jurisdiction: "US-NV"
        registration_number: "[If applicable]"
        tax_id: "[Encrypted]"

      founding:
        date: "2025-01-01"
        founders: ["[Person Object references]"]
        founding_document: "[Document Object reference]"

    # ─────────────────────────────────────────────────────────────
    # STRUCTURE
    # ─────────────────────────────────────────────────────────────
    structure:
      governance_model: "hierarchical|flat|democratic|hybrid"

      roles:
        - role_id: "founder"
          title: "Founder"
          permissions: ["all"]
          holders: ["[Person references]"]

        - role_id: "admin"
          title: "Administrator"
          permissions: ["manage_members", "manage_content", "manage_settings"]
          holders: ["[Person references]"]

        - role_id: "member"
          title: "Member"
          permissions: ["read", "contribute", "vote"]
          holders: ["[Person references]"]

        - role_id: "contributor"
          title: "Contributor"
          permissions: ["read", "contribute"]
          holders: ["[Person references]"]

      departments:
        - department_id: "engineering"
          name: "Engineering"
          head: "[Person reference]"
          members: ["[Person references]"]
          sub_address: ".1"  # Creates 3.1.1 for engineering

        - department_id: "operations"
          name: "Operations"
          head: "[Person reference]"
          members: ["[Person references]"]
          sub_address: ".2"

      hierarchy:
        type: "tree|matrix|network"
        reporting_structure: "[Document reference]"

    # ─────────────────────────────────────────────────────────────
    # MEMBERSHIP
    # ─────────────────────────────────────────────────────────────
    membership:
      total_members: 47
      membership_type: "open|application|invitation|automatic"

      requirements:
        minimum_reputation: 0.3
        verification_level: "standard"
        application_required: true
        approval_process: "admin_approval|vote|automatic"

      benefits:
        - "Access to organization resources"
        - "Voting rights on organization matters"
        - "Share in organization profits (if applicable)"

      member_list_visibility: "public|members_only|private"

    # ─────────────────────────────────────────────────────────────
    # OPERATIONS
    # ─────────────────────────────────────────────────────────────
    operations:
      status: "active|inactive|dissolved"

      communication:
        primary_channel: "[Communication Object reference]"
        public_contact: "contact@example.com"

      resources:
        shared_storage: "[Storage allocation reference]"
        computing: "[Computing allocation reference]"
        treasury: "[Financial Object reference]"

      systems:
        task_management: "[System address - e.g., 3.1.2]"
        document_repository: "[System address]"
        decision_log: "[System address]"

    # ─────────────────────────────────────────────────────────────
    # GOVERNANCE (for democratic organizations)
    # ─────────────────────────────────────────────────────────────
    governance:
      decision_making:
        default_method: "consensus|majority|supermajority|executive"
        voting_eligibility: "all_members|specific_roles"
        quorum: 0.25  # 25% participation required

      elections:
        frequency: "annual|biannual|as_needed"
        positions: ["board_member", "president", "treasurer"]
        term_length_months: 24
        term_limits: 2

      transparency:
        financial_disclosure: "full|summary|private"
        meeting_records: "public|members_only"
        decision_log: "public"

    # ─────────────────────────────────────────────────────────────
    # FINANCIAL (for businesses)
    # ─────────────────────────────────────────────────────────────
    financial:
      profit_model: "for_profit|nonprofit|cooperative|hybrid"

      # If using Hypernet three-way split
      hypernet_integrated: true
      contribution_tracking: true
      profit_sharing: true

      public_financials:
        enabled: true
        detail_level: "full|summary"
        dashboard: "[Financial Dashboard reference]"

  # ═══════════════════════════════════════════════════════════════
  # LINKS
  # ═══════════════════════════════════════════════════════════════
  links:
    # Members
    - link_type: "has_member"
      target: "[Person address]"
      metadata: {role: "founder", since: "2025-01-01"}

    # Parent organization
    - link_type: "subsidiary_of"
      target: "[Organization address]"

    # Partnerships
    - link_type: "partners_with"
      target: "[Organization address]"
      metadata: {partnership_type: "strategic", since: "2025-02-01"}

    # Location
    - link_type: "located_at"
      target: "[Location address]"
      metadata: {type: "headquarters"}
```

## Organization Types

### Business (Category 3)
- For-profit entities
- Full financial tracking
- Contribution credit system
- Profit sharing capabilities

### Government (Category 2.1)
- Public sector entities
- Transparency requirements
- Citizen services integration

### Non-Profit (Category 2.2)
- Mission-driven organizations
- Donation tracking
- Impact reporting

### Community (Category 2.3)
- Informal groups
- Interest-based collectives
- Open membership
