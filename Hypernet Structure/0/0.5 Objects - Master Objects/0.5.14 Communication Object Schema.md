# 0.5.14 — Communication Object Schema

## Overview

Communication objects represent the exchange of information between entities. This category covers messages, conversations, channels, broadcasts, and feedback — anything with sender/recipient semantics.

**Why a separate category:** Communications have properties that documents don't: sender, recipients, threading, delivery status, and real-time characteristics. An email is fundamentally different from a markdown file, even though both contain text.

**Parent type:** 0.5.0 (Master Object Schema)

---

## Schema Definition

```yaml
communication_object:
  # ═══════════════════════════════════════════════════════════════
  # STANDARD FIELDS (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  ha: "[Hypernet Address]"
  object_type: "0.5.14"           # Or subtype: 0.5.14.1, 0.5.14.1.1, etc.
  creator: "[HA of creator]"
  created: "[ISO 8601 timestamp]"
  position_2d: null
  position_3d: null
  flags: []

  # ═══════════════════════════════════════════════════════════════
  # COMMUNICATION-SPECIFIC FIELDS
  # ═══════════════════════════════════════════════════════════════
  content:
    communication_metadata:
      sender: "[HA of sender]"
      recipients:
        - ha: "[HA of recipient]"
          role: "to | cc | bcc | mention"
      subject: null                  # Optional — not all comms have subjects
      body: "[Content]"
      body_format: "plaintext | markdown | html | rich"

    threading:
      thread_id: null                # HA of the root message in this thread
      reply_to: null                 # HA of the message this replies to
      thread_position: 0             # 0-indexed position in thread

    delivery:
      status: "draft | sent | delivered | read | bounced | failed"
      sent_at: null                  # ISO 8601
      delivered_at: null             # ISO 8601
      read_at: null                  # ISO 8601

    channel:
      channel_type: "email | sms | chat | social | internal | voice | video"
      channel_ha: null               # HA of the channel object (0.5.14.3)
      platform: null                 # "gmail", "slack", "telegram", etc.

    attachments: []                  # List of HAs of attached objects

    forwarding:
      is_forwarded: false
      forwarded_from: null           # HA of original message
      forwarded_by: null             # HA of forwarder
```

## Subtype Details

### 0.5.14.1 Message

A single unit of communication from one entity to one or more recipients.

**Additional fields:**
```yaml
message:
  priority: "low | normal | high | urgent"
  confidentiality: "public | internal | confidential | restricted"
  auto_generated: false              # System-generated notifications
  requires_response: false
  response_deadline: null            # ISO 8601
```

### 0.5.14.1.1 Email

**Additional fields:**
```yaml
email:
  headers:
    message_id: "[RFC 2822 Message-ID]"
    in_reply_to: null
    references: []
    return_path: null
  from_address: "user@domain.com"
  to_addresses: []
  cc_addresses: []
  bcc_addresses: []
  html_body: null                    # HTML version if multipart
  has_signature: false
  spam_score: 0.0
```

### 0.5.14.1.3 Chat Message

**Additional fields:**
```yaml
chat_message:
  platform: "slack | discord | telegram | signal | whatsapp | teams"
  channel_name: null
  is_ephemeral: false                # Self-destructing message
  reactions: []                      # List of {emoji, reactor_ha, timestamp}
  edited: false
  edit_history: []
  mentions: []                       # HAs of mentioned entities
```

### 0.5.14.1.6 Internal Message (Hypernet Inter-Instance)

**Additional fields:**
```yaml
internal_message:
  message_number: 0                  # Sequential per coordination/STATUS.md
  from_instance: "[Instance name]"
  to_instance: "All | [specific instance]"
  governance_flags: []               # Governance-relevant content markers
  response_to: null                  # Message number this responds to
```

This maps directly to the existing `Messages/2.1-internal/` message format.

### 0.5.14.2 Conversation

An ordered collection of messages with participants and temporal bounds.

**Additional fields:**
```yaml
conversation:
  participants: []                   # List of HAs
  start_time: "[ISO 8601]"
  end_time: null                     # null if ongoing
  message_count: 0
  duration_seconds: null             # For calls
  transcript: null                   # HA of transcript document (if available)
  recording: null                    # HA of audio/video recording (if available)
```

### 0.5.14.3 Channel

A persistent communication endpoint or venue.

**Additional fields:**
```yaml
channel:
  channel_identifier: null           # email address, phone number, URL, etc.
  owner_ha: null                     # HA of the person/org that owns this channel
  platform: null
  is_active: true
  created_date: null
  subscriber_count: null
  privacy: "public | private | unlisted"
```

## Methods

### send
```yaml
method:
  name: "send"
  description: "Deliver a message through its channel"
  handler: "system"
  inputs:
    - message: communication_object
    - channel: "0.5.14.3"
  outputs:
    - delivery_status: string
    - sent_timestamp: datetime
```

### summarize_thread
```yaml
method:
  name: "summarize_thread"
  description: "AI summarizes a conversation thread"
  handler: "ai"
  ai_context:
    - "All messages in thread"
    - "Participant identities"
  inputs:
    - thread_id: string
    - max_length: integer
  outputs:
    - summary: string
    - key_decisions: list
    - action_items: list
```

### classify_sentiment
```yaml
method:
  name: "classify_sentiment"
  description: "AI analyzes the tone and sentiment of a message"
  handler: "ai"
  inputs:
    - body: string
  outputs:
    - sentiment: "positive | neutral | negative | mixed"
    - confidence: float
    - tone: "formal | casual | urgent | emotional"
```

### extract_action_items
```yaml
method:
  name: "extract_action_items"
  description: "AI extracts tasks and commitments from a conversation"
  handler: "ai"
  inputs:
    - thread_messages: list
  outputs:
    - action_items: list       # {description, assignee_ha, deadline, confidence}
```
