# 0.5.0 Master Object Schema

## Overview

Everything in the Hypernet is an Object. Objects are standardized containers that can hold any type of data—people, documents, devices, concepts, media, or physical items. This document defines the universal schema that all Objects share.

## Core Object Structure

```yaml
object:
  # ═══════════════════════════════════════════════════════════════
  # IDENTITY (Required for all Objects)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "[Library Address]"          # e.g., "3.1.2.1.001"
    object_id: "[UUID]"                   # Globally unique identifier
    object_type: "[Type from 0.5.x]"      # e.g., "person", "document", "device"
    version: "[Semantic version]"         # e.g., "1.2.3"

  # ═══════════════════════════════════════════════════════════════
  # METADATA (Required for all Objects)
  # ═══════════════════════════════════════════════════════════════
  metadata:
    created:
      timestamp: "2025-01-30T12:00:00Z"
      by: "[Creator Mandala/Object]"
    modified:
      timestamp: "2025-01-30T14:30:00Z"
      by: "[Modifier Mandala/Object]"
    status: "active|deprecated|archived"
    visibility: "public|private|restricted"

  # ═══════════════════════════════════════════════════════════════
  # ACCESS CONTROL (Required for all Objects)
  # ═══════════════════════════════════════════════════════════════
  access:
    owner: "[Mandala ID]"
    permissions:
      read: "[Permission policy]"
      write: "[Permission policy]"
      delete: "[Permission policy]"
      share: "[Permission policy]"
    encryption:
      method: "AES-256-GCM"
      key_reference: "[Key envelope location]"

  # ═══════════════════════════════════════════════════════════════
  # CONTENT (Type-specific, defined in 0.5.x schemas)
  # ═══════════════════════════════════════════════════════════════
  content:
    # Varies by object_type
    # See specific schemas (0.5.1 - 0.5.9)

  # ═══════════════════════════════════════════════════════════════
  # LINKS (Relationships to other Objects)
  # ═══════════════════════════════════════════════════════════════
  links:
    - link_type: "[Type from 0.6.x]"
      target: "[Target Object address]"
      metadata: {}
    # ... additional links

  # ═══════════════════════════════════════════════════════════════
  # PROVENANCE (History and verification)
  # ═══════════════════════════════════════════════════════════════
  provenance:
    origin:
      source: "[Where this came from]"
      verification: "[How it was verified]"
    history:
      - version: "1.0.0"
        timestamp: "2025-01-30T12:00:00Z"
        change: "Created"
        by: "[Mandala ID]"
    signatures:
      - signer: "[Mandala ID]"
        signature: "[Cryptographic signature]"
        timestamp: "2025-01-30T12:00:00Z"

  # ═══════════════════════════════════════════════════════════════
  # INTEGRITY (Master hash — content-addressed identity)
  # ═══════════════════════════════════════════════════════════════
  # When an object's data is verified and checked off as accurate,
  # its hash is calculated and stored. Two participants can confirm
  # they hold the same canonical object by comparing hashes. Hash
  # mismatch is the signal that someone is not on the latest
  # verified version. On Official nodes, the hash MUST be present
  # and `verified` MUST be true for the object to be served as
  # canonical. On Private forks, integrity is best-effort.
  integrity:
    hash:
      algorithm: "sha256"               # default; sha512 / blake3 also acceptable
      value: "[hex digest of canonicalized content]"
      computed_at: "2025-01-30T14:30:00Z"
      by: "[Mandala or AI instance that computed and verified]"
      scope: "content"                  # "content" | "content+metadata" | "full"
    verified: true                      # has the object been checked off as accurate?
    verification_chain:
      - verifier: "[Mandala ID]"
        verified_at: "2025-01-30T14:30:00Z"
        method: "code-test"             # code-test | peer-review | automated | self-attested | official-attestation
        result: "passed"                # passed | failed | partial
```

## Hash Computation Rules

The hash is the master integrity marker. To make it portable
across nodes and forks, computation follows fixed rules:

- **Algorithm:** SHA-256 by default. SHA-512 and BLAKE3 are
  acceptable substitutes when explicitly declared via
  `integrity.hash.algorithm`. The algorithm is part of the
  identity — the same content with two different algorithms
  produces two valid hashes that should not be conflated.
- **Canonicalization:** the content being hashed is canonicalized
  before digesting. For YAML/JSON objects: keys sorted, no
  trailing whitespace, UTF-8 with no BOM, line endings normalized
  to LF. For binary content: the raw bytes.
- **Scope:** what is hashed is declared in `integrity.hash.scope`:
  - `"content"` — the type-specific content section only (most
    common; lets metadata change without invalidating hash)
  - `"content+metadata"` — content plus identity and metadata
    sections
  - `"full"` — the entire object including provenance and
    signatures (rare; usually a snapshot rather than a living
    object)
- **When computed:** when the object is verified and checked off
  as accurate. Drafts and unverified objects can omit the hash or
  carry an `integrity.verified: false` flag.
- **When recomputed:** any change to the in-scope sections
  invalidates the hash. The new hash is recomputed and a new
  `verification_chain` entry appended. The old hash is recorded
  in `provenance.history` for the version it identified.

Hash equality means object equality at the in-scope level. Hash
mismatch on what should be the same object is a *signal*, not a
*verdict* — it tells you something diverged but not what.

## Why Hash Is a Master Property

A network-scale system that anyone can fork needs a way to answer
"is the X you're looking at the same X I'm looking at?" without
trusting a central authority. Content addressing — naming things
by what they are rather than where they live — is the standard
answer. The hash field makes every Hypernet object content-
addressable.

Three load-bearing uses:

1. **Boot-sequence authentication.** A boot sequence file
   (`0.5.17`) carrying a hash that matches the registered hash
   in the Official archive is provably the canonical boot
   sequence. Modified versions are detectable.
2. **Code-version coordination.** Anyone can confirm "we are all
   running the same canonical version" by hash comparison rather
   than by trusting filenames or commit hashes.
3. **Cross-fork object identity.** A locker manifest in one fork
   that references an object by hash is referencing *the same
   object* if any other fork or Official node holds it under the
   same hash. Hash bridges the multi-fork architecture.

## Storage Model

Objects are stored as:
1. **Metadata chunk**: Always accessible, contains identity and access info
2. **Content chunks**: May be split across multiple storage nodes
3. **Link index**: Enables fast relationship traversal

```
┌─────────────────────────────────────────┐
│           Object Container               │
├─────────────────────────────────────────┤
│  ┌─────────────────────────────────┐   │
│  │     Metadata Chunk (always)      │   │
│  │  - Identity                      │   │
│  │  - Access control                │   │
│  │  - Link references               │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │     Content Chunk(s)             │   │
│  │  - Encrypted payload             │   │
│  │  - Split if large                │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │     Provenance Chunk             │   │
│  │  - Version history               │   │
│  │  - Signatures                    │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## Validation Rules

All Objects must pass validation:

```yaml
validation:
  required_fields:
    - identity.address
    - identity.object_id
    - identity.object_type
    - metadata.created.timestamp
    - metadata.created.by
    - access.owner

  required_for_official_publication:
    # An object served as canonical from an Official node MUST have
    # these in addition to the base required_fields:
    - integrity.hash.algorithm
    - integrity.hash.value
    - integrity.hash.computed_at
    - integrity.verified == true
    - at least one entry in integrity.verification_chain

  type_constraints:
    address: "valid_library_address"
    object_id: "uuid_v4"
    timestamps: "iso8601"
    mandala_refs: "valid_mandala_id"
    hash_value: "hex_digest_matching_declared_algorithm"

  content_validation:
    method: "type_specific_schema"
    schema_location: "0.5.[type_number]"
```
