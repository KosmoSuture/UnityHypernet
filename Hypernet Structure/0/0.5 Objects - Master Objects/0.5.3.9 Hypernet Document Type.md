# 0.5.3.9 — Hypernet Document Type

## Overview

A Hypernet Document is a **compound object** — a document that assembles multiple objects (text blocks, images, videos, 3D models, or any other Hypernet object) into spatial arrangements for both 2D and 3D display.

This is how documents transition from flat text into spatial, immersive experiences. Every Hypernet Document specifies BOTH a 2D layout (for screens) and a 3D layout (for VR/AR), ensuring accessibility for all users regardless of their display hardware.

**Parent type:** 0.5.3 (Document)

---

## Schema Definition

```yaml
hypernet_document:
  # ═══════════════════════════════════════════════════════════════
  # STANDARD FIELDS (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  ha: "[Hypernet Address]"
  object_type: "0.5.3.9"
  creator: "[HA of creator]"
  created: "[ISO 8601 timestamp]"
  position_2d: null
  position_3d: null
  flags: []

  # ═══════════════════════════════════════════════════════════════
  # DOCUMENT METADATA
  # ═══════════════════════════════════════════════════════════════
  content:
    title: "Document Title"
    description: "Brief description of the document"
    version: "1.0.0"

  # ═══════════════════════════════════════════════════════════════
  # CANVAS — defines the space the document lives in
  # ═══════════════════════════════════════════════════════════════
  canvas:
    size_2d:
      width: 1920
      height: 1080
      unit: "px"
    size_3d:
      width: 19.2
      height: 10.8
      depth: 5.0
      unit: "m"                # Meters (real-world scale for VR)
    background: "#ffffff"
    grid_snap: true
    grid_size: 10              # Snap grid in 2D pixels

  # ═══════════════════════════════════════════════════════════════
  # ELEMENTS — the objects that make up the document
  # ═══════════════════════════════════════════════════════════════
  elements:
    - element_id: "e1"
      object_ref: "1.1.1.1.00001"   # Reference to existing Hypernet object
      object_type: "0.5.4.1.1"      # Cached type for fast rendering
      position_2d: {x: 100, y: 200}
      position_3d: {x: 1.0, y: 2.0, z: 0.0}
      size_2d: {width: 400, height: 300}
      size_3d: {width: 4.0, height: 3.0, depth: 0.01}
      rotation_2d: 0                 # Degrees
      rotation_3d: {x: 0, y: 0, z: 0}  # Euler angles
      layer: 0                       # Z-ordering in 2D view
      opacity: 1.0
      interactive: true              # Can be clicked/activated
      animation: null                # Optional animation definition

    - element_id: "e2"
      object_ref: null               # Inline content (no separate node)
      object_type: "0.5.3.1"         # Inline markdown text
      inline_content:
        type: "text"
        body: "This is a caption for the photo above."
        style:
          font_size: 14
          font_family: "system-ui"
          color: "#333333"
          text_align: "center"
      position_2d: {x: 100, y: 520}
      position_3d: {x: 1.0, y: 5.2, z: 0.0}
      size_2d: {width: 400, height: 40}
      size_3d: {width: 4.0, height: 0.4, depth: 0.01}
      layer: 1

    - element_id: "e3"
      object_ref: "1.1.2.1.00003"   # Reference to a video
      object_type: "0.5.4.2.1"      # MP4 video
      position_2d: {x: 550, y: 200}
      position_3d: {x: 5.5, y: 2.0, z: 0.5}
      size_2d: {width: 640, height: 360}
      size_3d: {width: 6.4, height: 3.6, depth: 0.01}
      layer: 0
      interactive: true
      autoplay: false

  # ═══════════════════════════════════════════════════════════════
  # NAVIGATION — how users move through the document
  # ═══════════════════════════════════════════════════════════════
  navigation:
    mode_2d: "scroll"              # "scroll" | "paginated" | "freeform"
    mode_3d: "walk"                # "walk" | "fly" | "teleport" | "guided"
    entry_point_3d: {x: 0, y: 1.7, z: -5.0}   # Where VR user starts
    pages: []                      # Optional page boundaries for paginated mode
    guided_path: []                # Optional waypoints for guided 3D tours
```

## Element Types

An element can reference any Hypernet object or contain inline content:

### Referenced Elements
Point to an existing object by HA. The renderer fetches the object and displays it according to its type definition.

```yaml
- object_ref: "1.1.1.1.00001"     # HA of the object
  object_type: "0.5.4.1.1"        # Type cached for rendering
```

### Inline Elements
Small content that doesn't warrant its own node (captions, labels, dividers):

```yaml
- object_ref: null
  inline_content:
    type: "text | shape | divider"
    body: "Content here"
    style: {}
```

## Methods

### render_2d
```yaml
method:
  name: "render_2d"
  description: "Render the document as a 2D layout (web, print, screen)"
  handler: "system"
  inputs:
    - document: hypernet_document
    - viewport: {width, height}
  outputs:
    - html: string               # Rendered HTML/CSS
    - assets: list                # Referenced objects to fetch
```

### render_3d
```yaml
method:
  name: "render_3d"
  description: "Render the document as a 3D scene (VR/AR)"
  handler: "system"
  inputs:
    - document: hypernet_document
    - renderer: "webxr | unity | unreal"
  outputs:
    - scene_graph: dict           # 3D scene description
    - assets: list                # Referenced objects to fetch
```

### compose
```yaml
method:
  name: "compose"
  description: "AI arranges objects into a coherent document layout"
  handler: "ai"
  ai_context:
    - "List of objects to include"
    - "Document purpose and audience"
    - "Available canvas dimensions"
  inputs:
    - objects: list[string]       # HAs of objects to arrange
    - purpose: string             # "presentation", "report", "gallery", etc.
    - style: string               # "minimal", "rich", "immersive"
  outputs:
    - elements: list              # Positioned elements
    - canvas: dict                # Computed canvas dimensions
```

## Design Principles

1. **Dual representation** — every document has BOTH 2D and 3D layouts, specified explicitly
2. **Object composition** — documents are lists of references, not copies of content
3. **Type-aware rendering** — each element's object_type tells the renderer how to display it
4. **AI-composable** — AI workers can automatically arrange objects into documents
5. **Accessible** — 2D view is always available for non-VR users
