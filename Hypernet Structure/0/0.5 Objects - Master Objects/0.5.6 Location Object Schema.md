# 0.5.6 Location Object Schema

## Overview

Location Objects represent places—both physical locations in the real world and virtual spaces within the Hypernet VR environment. They serve as anchors for spatial organization of information and enable location-based services.

## Schema Definition

```yaml
location_object:
  # ═══════════════════════════════════════════════════════════════
  # BASE OBJECT (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "8.[category].[instance]"
    object_id: "[UUID]"
    object_type: "location"
    subtype: "physical|virtual|hybrid"
    version: "1.0.0"

  # ═══════════════════════════════════════════════════════════════
  # LOCATION-SPECIFIC CONTENT
  # ═══════════════════════════════════════════════════════════════
  content:
    # ─────────────────────────────────────────────────────────────
    # LOCATION IDENTITY
    # ─────────────────────────────────────────────────────────────
    location_identity:
      name: "Location Name"
      description: "Description of this location"
      aliases: ["Other names", "Local names"]

      type:
        primary: "building|room|outdoor|region|virtual_space"
        secondary: "office|residence|park|city|vr_environment"

      images:
        primary: "[Media Object reference]"
        gallery: ["[Media references]"]

    # ─────────────────────────────────────────────────────────────
    # PHYSICAL COORDINATES (for physical locations)
    # ─────────────────────────────────────────────────────────────
    physical:
      coordinates:
        latitude: 36.1699
        longitude: -115.1398
        altitude_meters: 610
        accuracy_meters: 5

      address:
        formatted: "123 Main Street, Las Vegas, NV 89101, USA"
        components:
          street_number: "123"
          street_name: "Main Street"
          unit: "Suite 456"
          city: "Las Vegas"
          state: "Nevada"
          postal_code: "89101"
          country: "United States"
          country_code: "US"

      boundaries:
        type: "polygon|circle|box"
        # For polygon
        coordinates: [[lat1, lon1], [lat2, lon2], ...]
        # For circle
        center: {lat: 36.1699, lon: -115.1398}
        radius_meters: 100

      timezone: "America/Los_Angeles"

      what3words: "filled.count.soap"  # Optional alternative addressing

    # ─────────────────────────────────────────────────────────────
    # VIRTUAL SPACE (for VR locations)
    # ─────────────────────────────────────────────────────────────
    virtual:
      space_type: "room|building|landscape|abstract"

      environment:
        template: "[Environment template reference]"
        customizations: {}

      dimensions:
        width_meters: 20
        depth_meters: 30
        height_meters: 5
        unlimited: false

      spawn_point:
        position: {x: 0, y: 0, z: 5}
        rotation: {x: 0, y: 180, z: 0}

      navigation:
        walkable_area: "[Mesh reference]"
        teleport_points: [
          {name: "entrance", position: {x: 0, y: 0, z: 0}},
          {name: "main_display", position: {x: 10, y: 0, z: 15}}
        ]

      physics:
        gravity: 9.81
        collision_enabled: true

      capacity:
        recommended: 20
        maximum: 100

    # ─────────────────────────────────────────────────────────────
    # CONTAINED OBJECTS
    # ─────────────────────────────────────────────────────────────
    contents:
      # Objects placed at this location
      placed_objects:
        - object: "[Object reference]"
          position: {x: 5, y: 1, z: 10}
          rotation: {x: 0, y: 45, z: 0}
          scale: 1.0

      # Information displays
      information_points:
        - name: "Welcome Panel"
          type: "text_display"
          position: {x: 0, y: 1.5, z: 1}
          content: "[Document reference]"

      # Sub-locations
      sub_locations:
        - sub_location: "[Location reference]"
          relative_position: {x: 10, y: 0, z: 0}
          entrance: {x: 10, y: 0, z: -1}

    # ─────────────────────────────────────────────────────────────
    # ACCESS AND HOURS
    # ─────────────────────────────────────────────────────────────
    access:
      visibility: "public|private|restricted"

      physical_access:
        public_access: true
        accessibility:
          wheelchair: true
          elevator: true
          braille: false
          audio_guide: true

        hours:
          monday: {open: "09:00", close: "17:00"}
          tuesday: {open: "09:00", close: "17:00"}
          # ... other days
          exceptions:
            - date: "2025-12-25"
              status: "closed"
              reason: "Holiday"

      virtual_access:
        public: true
        requires_invitation: false
        age_restriction: null
        content_rating: "G"

    # ─────────────────────────────────────────────────────────────
    # REAL-TIME STATUS
    # ─────────────────────────────────────────────────────────────
    status:
      operational: true
      current_conditions:
        weather: {temp_c: 22, conditions: "sunny"}  # For outdoor
        occupancy: 45  # Current people count
        noise_level: "moderate"
        air_quality: "good"

      alerts:
        - type: "closure"
          message: "Elevator maintenance until 3pm"
          expires: "2025-01-30T15:00:00Z"

    # ─────────────────────────────────────────────────────────────
    # OWNERSHIP AND MANAGEMENT
    # ─────────────────────────────────────────────────────────────
    management:
      owner: "[Person/Organization reference]"
      operator: "[Organization reference]"
      contact: "[Person reference]"

      rules:
        - "No photography"
        - "Quiet hours after 10pm"

  # ═══════════════════════════════════════════════════════════════
  # LINKS
  # ═══════════════════════════════════════════════════════════════
  links:
    # Ownership
    - link_type: "owned_by"
      target: "[Person/Organization address]"

    # Hierarchy
    - link_type: "part_of"
      target: "[Larger Location address]"
      metadata: {relationship: "room_in_building"}

    # Contains
    - link_type: "contains"
      target: "[Sub-Location address]"

    # Events here
    - link_type: "hosts"
      target: "[Event address]"

    # Associated with
    - link_type: "associated_with"
      target: "[Organization address]"
      metadata: {type: "headquarters"}
```

## Location Categories

### Physical Locations
Real-world places with GPS coordinates and addresses.

#### Types
- **Buildings**: Offices, homes, stores, institutions
- **Rooms**: Specific spaces within buildings
- **Outdoor**: Parks, plazas, natural areas
- **Regions**: Cities, neighborhoods, countries

### Virtual Locations
VR spaces within the Hypernet.

#### Types
- **Personal Spaces**: User homes, offices
- **Public Spaces**: Galleries, meeting halls, parks
- **Functional Spaces**: Libraries, workspaces
- **Abstract Spaces**: Information visualizations

### Hybrid Locations
Physical places with virtual overlays or digital twins.

## VR Navigation

Locations in VR can be:
- **Walked to**: If connected via paths
- **Teleported to**: Instant travel
- **Portaled**: Transition through doorways
