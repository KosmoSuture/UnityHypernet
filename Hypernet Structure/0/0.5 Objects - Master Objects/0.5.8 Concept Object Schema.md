# 0.5.8 Concept Object Schema

## Overview

Concept Objects represent abstract ideas, categories, tags, theories, and relationship types. They form the semantic backbone of the Hypernet, enabling meaningful organization and discovery of information through structured knowledge representation.

## Schema Definition

```yaml
concept_object:
  # ═══════════════════════════════════════════════════════════════
  # BASE OBJECT (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "9.[category].[instance]"
    object_id: "[UUID]"
    object_type: "concept"
    subtype: "category|tag|theory|relationship_type|definition"
    version: "1.0.0"

  # ═══════════════════════════════════════════════════════════════
  # CONCEPT-SPECIFIC CONTENT
  # ═══════════════════════════════════════════════════════════════
  content:
    # ─────────────────────────────────────────────────────────────
    # CONCEPT IDENTITY
    # ─────────────────────────────────────────────────────────────
    concept_identity:
      name: "Concept Name"
      definition: "Clear, precise definition of this concept"
      short_definition: "Brief one-line definition"

      aliases:
        - term: "Alternative Name"
          language: "en"
          context: "informal"
        - term: "Término Alternativo"
          language: "es"

      scope: "universal|domain_specific|local"
      domain: "technology|science|business|philosophy|etc"

    # ─────────────────────────────────────────────────────────────
    # SEMANTIC PROPERTIES
    # ─────────────────────────────────────────────────────────────
    semantics:
      # For categories (taxonomic concepts)
      category:
        parent: "[Parent Concept reference]"
        children: ["[Child Concept references]"]
        level: 2  # Depth in hierarchy
        is_leaf: false

      # For tags (folksonomy)
      tag:
        usage_count: 15234
        trending: true
        related_tags: ["[Tag references]"]

      # For theories/models
      theory:
        domain: "physics|economics|psychology|etc"
        status: "hypothesis|theory|law|disputed"
        evidence_strength: 0.85
        key_papers: ["[Document references]"]

      # For relationship types (used in Links)
      relationship_type:
        source_types: ["person", "organization"]
        target_types: ["person", "organization"]
        cardinality: "one_to_one|one_to_many|many_to_many"
        symmetric: false
        transitive: false
        inverse: "[Inverse relationship Concept reference]"

    # ─────────────────────────────────────────────────────────────
    # KNOWLEDGE REPRESENTATION
    # ─────────────────────────────────────────────────────────────
    knowledge:
      # Formal definition (for machine processing)
      formal:
        notation: "first_order_logic|description_logic|owl|custom"
        expression: "∀x(Human(x) → Mortal(x))"

      # Natural language explanations
      explanations:
        - level: "expert"
          text: "Technical explanation for domain experts"
        - level: "intermediate"
          text: "Explanation for educated general audience"
        - level: "beginner"
          text: "Simple explanation for newcomers"
        - level: "child"
          text: "Explanation a child could understand"

      # Examples
      examples:
        - example: "Concrete example of this concept"
          context: "When this applies"
        - counter_example: "What this is NOT"
          explanation: "Why this doesn't fit"

      # Distinguishing features
      differentia:
        - "What makes this different from similar concepts"
        - "Key distinguishing characteristic"

    # ─────────────────────────────────────────────────────────────
    # ONTOLOGICAL PROPERTIES
    # ─────────────────────────────────────────────────────────────
    ontology:
      # Classification in formal ontologies
      upper_ontology:
        sumo: "Process"  # Suggested Upper Merged Ontology
        dolce: "perdurant"  # Descriptive Ontology for Linguistic and Cognitive Engineering
        bfo: "occurrent"  # Basic Formal Ontology

      # Philosophical category
      metaphysical_category: "abstract|concrete|process|property|relation"

      # Existence conditions
      existence:
        mind_dependent: true  # Exists only in minds
        time_dependent: false  # Exists across all time
        observer_dependent: false

    # ─────────────────────────────────────────────────────────────
    # USAGE AND GOVERNANCE
    # ─────────────────────────────────────────────────────────────
    governance:
      # Who controls this concept definition
      authority:
        type: "official|community|individual"
        maintainer: "[Person/Organization reference]"

      # Standards alignment
      standards:
        - standard: "ISO 12345"
          mapping: "equivalent"
        - standard: "W3C SKOS"
          uri: "http://example.org/skos/concept"

      # Change control
      stability: "stable|evolving|experimental"
      deprecation_notice: null  # Or deprecation info

    # ─────────────────────────────────────────────────────────────
    # VISUALIZATION
    # ─────────────────────────────────────────────────────────────
    visualization:
      icon: "[Media reference]"
      color: "#4A90D9"

      # For VR representation
      vr_representation:
        model: "[3D Model reference]"
        abstract_visualization: "sphere|cube|network|custom"
        size_represents: "importance|usage_count|none"

    # ─────────────────────────────────────────────────────────────
    # MULTILINGUAL
    # ─────────────────────────────────────────────────────────────
    translations:
      - language: "es"
        name: "Nombre del Concepto"
        definition: "Definición en español"
      - language: "zh"
        name: "概念名称"
        definition: "中文定义"
      - language: "ar"
        name: "اسم المفهوم"
        definition: "التعريف بالعربية"

  # ═══════════════════════════════════════════════════════════════
  # LINKS
  # ═══════════════════════════════════════════════════════════════
  links:
    # Hierarchical
    - link_type: "broader_than"
      target: "[Narrower Concept address]"

    - link_type: "narrower_than"
      target: "[Broader Concept address]"

    # Associative
    - link_type: "related_to"
      target: "[Related Concept address]"
      metadata: {relationship: "similar"}

    # Definitional
    - link_type: "defined_by"
      target: "[Document address]"
      metadata: {authoritative: true}

    # Instance
    - link_type: "has_instance"
      target: "[Object address]"

    # Contrast
    - link_type: "opposite_of"
      target: "[Opposing Concept address]"
```

## Concept Types

### Categories
Hierarchical classification nodes for organizing objects.

### Tags
Flat, user-generated labels for flexible organization.

### Theories
Scientific or philosophical frameworks.

### Relationship Types
Definitions of link types used between objects.

### Definitions
Formal term definitions and glossary entries.

## Role in Hypernet

Concepts provide:
- **Taxonomy**: Hierarchical organization
- **Folksonomy**: User-driven tagging
- **Ontology**: Formal knowledge representation
- **Discovery**: Semantic search and navigation
- **Interoperability**: Standard vocabulary
