# 0.5.11 — Financial Object Schema

## Overview

Financial objects represent monetary value, instruments, transactions, and economic activity. This category covers everything from a coffee purchase to a stock portfolio to a national budget.

**Why a separate category:** Financial objects have a unique structural signature: amount, currency, counterparty, date. No other object type has these as essential properties. Forcing financial data into Event or Document loses the semantic precision needed for queries like "What did I spend on groceries last month?"

**Parent type:** 0.5.0 (Master Object Schema)

---

## Schema Definition

```yaml
financial_object:
  # ═══════════════════════════════════════════════════════════════
  # STANDARD FIELDS (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  ha: "[Hypernet Address]"
  object_type: "0.5.11"           # Or subtype: 0.5.11.1, 0.5.11.2.1, etc.
  creator: "[HA of creator]"
  created: "[ISO 8601 timestamp]"
  position_2d: null
  position_3d: null
  flags: []

  # ═══════════════════════════════════════════════════════════════
  # FINANCIAL-SPECIFIC FIELDS
  # ═══════════════════════════════════════════════════════════════
  content:
    financial_metadata:
      amount: 0.00                   # Decimal, never floating point for money
      currency: "USD"                # ISO 4217 currency code
      amount_minor_units: 0          # Amount in smallest currency unit (cents)
      exchange_rate: null            # If converted from another currency

    parties:
      payer: null                    # HA of entity paying
      payee: null                    # HA of entity receiving
      intermediary: null             # HA of bank, processor, etc.

    timing:
      transaction_date: null         # When the transaction occurred
      settlement_date: null          # When money actually moved
      effective_date: null           # When it takes effect (for contracts)
      period_start: null             # For recurring (budget period, billing cycle)
      period_end: null

    classification:
      category: null                 # "groceries", "salary", "rent", etc.
      subcategory: null
      tax_category: null             # "deductible", "taxable_income", "exempt"
      is_recurring: false
      recurrence_rule: null          # iCal RRULE format

    source:
      institution: null              # HA of bank, exchange, etc.
      account_ha: null               # HA of the account (0.5.11.1)
      external_id: null              # Transaction ID from the source system
      import_source: null            # "plaid", "csv_import", "manual", etc.
```

## Subtype Details

### 0.5.11.1 Account

A container for financial value — tracks balance and transaction history.

**Additional fields:**
```yaml
account:
  account_name: "Checking Account"
  account_number_masked: "****4532"  # Never store full account numbers in cleartext
  institution_name: "Chase Bank"
  institution_ha: null               # HA of the organization
  account_type: "checking | savings | credit | investment | crypto | loan"
  balance: 0.00
  balance_currency: "USD"
  balance_as_of: null                # ISO 8601
  credit_limit: null                 # For credit accounts
  interest_rate: null                # APR as decimal (0.0425 = 4.25%)
  opened_date: null
  closed_date: null
  is_active: true
```

### 0.5.11.2 Transaction

A single movement of value. The most common financial object.

**Additional fields:**
```yaml
transaction:
  description: "Grocery Store Purchase"
  merchant_name: null
  merchant_ha: null                  # HA of the merchant (Organization)
  transaction_type: "purchase | income | transfer | fee | refund"
  payment_method: "debit | credit | cash | check | wire | crypto | ach"
  status: "pending | posted | cleared | reversed | failed"
  location_ha: null                  # Where the transaction occurred
  receipt_ha: null                   # HA of receipt image/document
  notes: null                        # User notes
  tags: []                           # User-defined tags
```

### 0.5.11.3 Instrument

A financial product that holds or represents value.

**Additional fields:**
```yaml
instrument:
  instrument_type: "equity | bond | option | derivative | token | insurance"
  ticker_symbol: null                # "AAPL", "BTC", etc.
  issuer: null                       # HA of issuing organization
  face_value: null
  market_value: null
  market_value_as_of: null
  maturity_date: null                # For bonds
  strike_price: null                 # For options
  expiration_date: null
  quantity: 0                        # Shares, units, tokens
  cost_basis: null                   # Original purchase price
```

### 0.5.11.4 Budget

A plan for allocating financial resources over a time period.

**Additional fields:**
```yaml
budget:
  period: "monthly | quarterly | annual | project"
  total_planned: 0.00
  total_actual: 0.00
  categories:
    - name: "Category Name"
      planned: 0.00
      actual: 0.00
      variance: 0.00                # actual - planned
  owner_ha: null                     # Person or Organization
  approval_status: "draft | proposed | approved | expired"
```

## Privacy & Security Notes

Financial data is among the most sensitive in the Hypernet:

1. **Never store full account numbers** — use masked versions (`****4532`)
2. **Full account numbers** should be stored only in the Gen 1 `access.encryption` section when implemented
3. **Transaction data** should default to `visibility: "private"`
4. **Tax documents** should be flagged `0.8.2.sensitive`
5. **Compliance:** Financial data may be subject to GLBA, PCI-DSS, or regional regulations. The `flags` field should indicate applicable compliance requirements.

## Mapping from 0.4 Object Type Registry

The 0.4 registry planned but did not implement Financial Types (listed as "Future" at 0.0.5):
- Transaction → 0.5.11.2 Transaction
- BankAccount → 0.5.11.1.1 Bank Account
- Investment → 0.5.11.3 Instrument
- Bill → 0.5.11.5.3 Bill
- Receipt → 0.5.11.5.2 Receipt
- Budget → 0.5.11.4 Budget

This schema provides the formal definitions the registry was waiting for.
