# 0.5.1 Person Object Schema

## Overview

Person Objects represent individual humans in the Hypernet. They are the foundation of identity, privacy, and participation. Every Person Object is controlled by its owner through their Mandala.

## Schema Definition

```yaml
person_object:
  # ═══════════════════════════════════════════════════════════════
  # BASE OBJECT (inherited from 0.5.0)
  # ═══════════════════════════════════════════════════════════════
  identity:
    address: "1.[region].[instance]"
    object_id: "[UUID]"
    object_type: "person"
    version: "1.0.0"

  metadata:
    created:
      timestamp: "2025-01-30T12:00:00Z"
      by: "[Self - Mandala ID]"
    # ... standard metadata

  access:
    owner: "[Own Mandala ID]"
    # Person objects are always self-owned

  # ═══════════════════════════════════════════════════════════════
  # PERSON-SPECIFIC CONTENT
  # ═══════════════════════════════════════════════════════════════
  content:
    # ─────────────────────────────────────────────────────────────
    # PUBLIC PROFILE (visible to all)
    # ─────────────────────────────────────────────────────────────
    public_profile:
      display_name: "Optional public name"
      avatar: "[Media Object reference]"
      bio: "Short public biography"
      public_links:
        - type: "website"
          url: "https://example.com"
        - type: "social"
          platform: "twitter"
          handle: "@example"

      reputation:
        score: 0.75
        level: "trusted"
        badges: ["contributor", "early_adopter"]

      public_interests: ["technology", "education", "music"]

      joined_date: "2025-01-01"

    # ─────────────────────────────────────────────────────────────
    # LOCKERS (Private, segmented data)
    # ─────────────────────────────────────────────────────────────
    lockers:
      # Each locker has its own access policy

      identity_locker:
        access_policy: "[Strict - legal requirements only]"
        contents:
          legal_name: "[Encrypted]"
          date_of_birth: "[Encrypted]"
          government_ids: "[Encrypted references]"
          citizenship: "[Encrypted]"

      contact_locker:
        access_policy: "[Selective sharing]"
        contents:
          email_addresses: "[Encrypted list]"
          phone_numbers: "[Encrypted list]"
          physical_addresses: "[Encrypted list]"

      financial_locker:
        access_policy: "[Very strict]"
        contents:
          payment_methods: "[Encrypted references]"
          tax_information: "[Encrypted]"
          contribution_credits: 12500

      health_locker:
        access_policy: "[Healthcare providers only]"
        contents:
          medical_records: "[Encrypted references]"
          insurance: "[Encrypted]"
          emergency_contacts: "[Encrypted]"

      professional_locker:
        access_policy: "[Business connections]"
        contents:
          resume: "[Document Object reference]"
          skills: ["skill1", "skill2", "skill3"]
          certifications: ["[Document references]"]
          work_history: ["[Organization references]"]

      social_locker:
        access_policy: "[Friends and connections]"
        contents:
          personal_interests: "[Detailed list]"
          relationship_status: "[Encrypted]"
          family_connections: "[Person references]"

    # ─────────────────────────────────────────────────────────────
    # MANDALA (Identity management)
    # ─────────────────────────────────────────────────────────────
    mandala:
      mandala_id: "[Primary identifier]"
      public_key: "[Ed25519 public key]"
      verification_level: "enhanced"
      recovery_methods:
        - type: "social_recovery"
          trustees: ["mandala1", "mandala2", "mandala3"]
          threshold: 2
        - type: "hardware_key"
          device: "[Device Object reference]"

    # ─────────────────────────────────────────────────────────────
    # PREFERENCES
    # ─────────────────────────────────────────────────────────────
    preferences:
      language: "en"
      timezone: "America/Los_Angeles"
      notification_settings:
        email: true
        push: true
        digest_frequency: "daily"
      privacy_defaults:
        new_content: "private"
        location_sharing: false
        activity_visibility: "connections_only"
      vr_preferences:
        avatar_style: "[3D Model reference]"
        default_environment: "[Location reference]"
        accessibility: {}

  # ═══════════════════════════════════════════════════════════════
  # LINKS
  # ═══════════════════════════════════════════════════════════════
  links:
    # Relationships to other people
    - link_type: "knows"
      target: "[Person address]"
      metadata: {since: "2024-01-01", relationship: "friend"}

    # Organizational affiliations
    - link_type: "member_of"
      target: "[Organization address]"
      metadata: {role: "contributor", since: "2025-01-01"}

    # Created content
    - link_type: "created"
      target: "[Document address]"
      metadata: {role: "author"}

    # Owned devices
    - link_type: "owns"
      target: "[Device address]"
      metadata: {acquired: "2025-01-15"}
```

## Privacy Architecture

### Locker Access Levels

| Level | Description | Example Use |
|-------|-------------|-------------|
| Private | Only owner | Personal notes |
| Selective | Specific individuals | Medical with doctor |
| Group | Organization members | Work contact info |
| Connections | Approved contacts | Social information |
| Public | Everyone | Display name, avatar |

### Zero-Knowledge Verification

Others can verify attributes without seeing data:
```
"Is this person over 18?" → true/false (no birthdate revealed)
"Does this person have certification X?" → true/false (no details)
"Is this person in region Y?" → true/false (no exact location)
```

## Verification Levels

| Level | Requirements | Unlocks |
|-------|--------------|---------|
| Basic | Email verification | Account creation |
| Standard | Email + phone | Basic participation |
| Enhanced | Government ID check | Full features |
| Full | Biometric + in-person | Governance eligibility |
| Vouched | Enhanced + 3 high-rep vouchers | Maximum trust |
