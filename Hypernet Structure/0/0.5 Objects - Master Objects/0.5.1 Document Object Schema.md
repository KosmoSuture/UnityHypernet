# 0.5.1 Document Object Schema

## Overview

Document Objects contain text-based content. They are the primary way information is stored and shared in the Hypernet, from simple notes to formal publications.

## Schema Definition

```yaml
document_object:
  # === BASE OBJECT (inherited from 0.5.0) ===
  _extends: "hypernet_object"
  object_type: "document"

  # === DOCUMENT-SPECIFIC CONTENT ===
  content:
    title: "Document title"
    subtitle: "Optional subtitle"

    body:
      format: "markdown|html|plaintext|structured"
      content: "[Document content]"
      sections: []  # Optional section breakdown

    abstract: "Brief summary (for searchability)"

    language: "en-US"  # ISO language code
    translations: []   # Links to translated versions

  # === DOCUMENT METADATA ===
  document_metadata:
    document_type: "[See Document Types below]"
    status: "draft|review|published|retracted"

    authorship:
      authors: ["[Person Object addresses]"]
      contributors: ["[Person Object addresses]"]
      editor: "[Person Object address]"

    publication:
      published_date: "2025-01-30"
      publisher: "[Organization Object address]"
      edition: "1st"
      doi: "[DOI if applicable]"

    classification:
      categories: ["category1", "category2"]
      tags: ["tag1", "tag2", "tag3"]
      keywords: ["keyword1", "keyword2"]

    licensing:
      license_type: "CC-BY-4.0|proprietary|public_domain|..."
      license_text: "[Full license or link]"
      attribution_required: true

  # === TRUTH/EVIDENCE (for factual claims) ===
  evidence:
    claim_type: "factual|opinion|creative|procedural"
    sources: ["[Source Object addresses]"]
    citations: []  # Inline citation references
    verification_status: "unverified|community_verified|expert_verified"
    evidence_score: 0.0-1.0  # Computed from sources and verification
    fact_check_results: []

  # === DOCUMENT STRUCTURE ===
  structure:
    word_count: 1234
    reading_time_minutes: 5
    table_of_contents: []
    figures: ["[Media Object addresses]"]
    tables: ["[Data Object addresses]"]
    appendices: ["[Document Object addresses]"]

  # === RELATIONSHIPS ===
  document_links:
    references: ["[Cited Document addresses]"]
    cited_by: ["[Documents citing this one]"]
    related_documents: ["[Related content]"]
    parent_document: "[If this is a section/chapter]"
    child_documents: ["[Sections/chapters]"]
    supersedes: "[Document this replaces]"
    superseded_by: "[Document replacing this]"
```

## Document Types

| Type | Description | Typical Use |
|------|-------------|-------------|
| article | Standalone written piece | Blog posts, news, essays |
| report | Formal documentation | Business reports, research |
| specification | Technical standards | API docs, protocols |
| manual | Instructional content | User guides, how-tos |
| policy | Rules and guidelines | Governance, procedures |
| contract | Binding agreements | Legal documents |
| proposal | Suggested changes | HEPs, project proposals |
| memo | Internal communication | Notes, announcements |
| transcript | Recorded speech | Meeting notes, interviews |
| creative | Artistic writing | Fiction, poetry |

## Evidence and Truth

### Claim Types
- **Factual**: Statements that can be verified (require sources)
- **Opinion**: Subjective viewpoints (attributed to author)
- **Creative**: Artistic expression (not truth-claims)
- **Procedural**: Instructions (verified by testing)

### Verification Levels
```yaml
verification_levels:
  unverified:
    description: "No verification performed"
    display: "Unverified claim"
    trust_multiplier: 0.5

  community_verified:
    description: "Checked by community members"
    requirements:
      - min_verifiers: 3
      - min_verifier_reputation: 0.5
    trust_multiplier: 0.8

  expert_verified:
    description: "Verified by domain experts"
    requirements:
      - min_verifiers: 2
      - expertise_match: true
      - min_expert_reputation: 0.8
    trust_multiplier: 1.0

  disputed:
    description: "Conflicting evidence exists"
    display: "Disputed claim - see discussion"
    trust_multiplier: 0.3
```

## Examples

### Simple Article
```yaml
address: "4.1.2345"
object_type: "document"
content:
  title: "Introduction to Hypernet"
  body:
    format: "markdown"
    content: "# Introduction\n\nThe Hypernet is..."
document_metadata:
  document_type: "article"
  status: "published"
  authorship:
    authors: ["1.1"]  # Matt Schaeffer
evidence:
  claim_type: "factual"
  verification_status: "expert_verified"
```

### Technical Specification
```yaml
address: "4.3.0.5.1"
object_type: "document"
content:
  title: "Document Object Schema"
  body:
    format: "structured"
    sections:
      - title: "Overview"
        content: "..."
      - title: "Schema Definition"
        content: "..."
document_metadata:
  document_type: "specification"
  status: "published"
  classification:
    categories: ["technical", "schema"]
```
