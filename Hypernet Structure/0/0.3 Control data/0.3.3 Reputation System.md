# 0.3.3 Reputation System

## Overview

The Hypernet reputation system creates accountability and trust without centralized authority. Reputation affects voting weight, access privileges, and economic participation.

## Reputation Score

### Score Range
- **Minimum**: 0.0 (new or severely penalized)
- **Maximum**: 1.0 (highest trust level)
- **Starting**: 0.2 (new verified participants)

### Score Components

| Component | Weight | Description |
|-----------|--------|-------------|
| Contribution | 30% | Value added to the network |
| Reliability | 25% | Consistency and dependability |
| Verification | 20% | Identity verification level |
| Community | 15% | Peer endorsements and interactions |
| Longevity | 10% | Time participating in network |

## Component Details

### 1. Contribution Score (30%)
```yaml
contribution_factors:
  code_contributions:
    weight: 0.3
    metrics: ["commits", "reviews", "issues_resolved"]

  content_creation:
    weight: 0.25
    metrics: ["objects_created", "quality_ratings", "citations"]

  governance_participation:
    weight: 0.2
    metrics: ["votes_cast", "proposals_submitted", "committee_service"]

  community_support:
    weight: 0.15
    metrics: ["questions_answered", "mentoring", "onboarding_help"]

  infrastructure:
    weight: 0.1
    metrics: ["node_operation", "uptime", "resources_provided"]
```

### 2. Reliability Score (25%)
```yaml
reliability_factors:
  commitment_fulfillment:
    weight: 0.4
    description: "Completing agreed tasks and obligations"

  response_consistency:
    weight: 0.3
    description: "Predictable, timely responses"

  accuracy:
    weight: 0.2
    description: "Correctness of provided information"

  availability:
    weight: 0.1
    description: "Being accessible when expected"
```

### 3. Verification Score (20%)
| Verification Level | Score Boost | Requirements |
|-------------------|-------------|--------------|
| Basic | 0.0 | Email verified |
| Standard | 0.3 | Phone + email verified |
| Enhanced | 0.6 | Government ID verified |
| Full | 0.8 | Biometric + in-person |
| Vouched | 1.0 | Enhanced + 3 high-rep vouchers |

### 4. Community Score (15%)
```yaml
community_factors:
  endorsements:
    weight: 0.4
    description: "Positive endorsements from peers"
    calculation: "weighted by endorser reputation"

  interactions:
    weight: 0.3
    description: "Quality of community interactions"
    metrics: ["helpful_ratings", "collaboration_success"]

  disputes:
    weight: 0.3
    description: "Dispute history"
    penalty_for: ["lost_disputes", "reported_violations"]
```

### 5. Longevity Score (10%)
```
longevity_score = min(1.0, account_age_months / 36)
```

## Reputation Levels

| Level | Score Range | Privileges |
|-------|-------------|------------|
| Probationary | 0.0 - 0.2 | Basic access, limited features |
| Member | 0.2 - 0.4 | Standard user capabilities |
| Contributor | 0.4 - 0.6 | Enhanced access, voting rights |
| Trusted | 0.6 - 0.8 | Moderation capabilities, weighted votes |
| Steward | 0.8 - 0.95 | Governance eligibility, high trust |
| Guardian | 0.95 - 1.0 | System stewardship, maximum trust |

## Reputation Changes

### Positive Actions
| Action | Impact | Cooldown |
|--------|--------|----------|
| Quality contribution | +0.001 to +0.01 | Per item |
| Successful proposal | +0.02 | 30 days |
| Dispute won (as defendant) | +0.01 | Per dispute |
| Peer endorsement | +0.005 | Per endorser per year |
| Node operation (monthly) | +0.002 | Monthly |

### Negative Actions
| Action | Impact | Recovery |
|--------|--------|----------|
| Content violation | -0.05 to -0.2 | Appeal process |
| Lost dispute | -0.02 to -0.1 | Based on severity |
| Commitment failure | -0.01 to -0.05 | Complete obligations |
| Spam/abuse | -0.1 to -0.5 | Review period |
| Fraud/deception | -0.5 to -1.0 | Permanent or lengthy ban |

### Decay
Reputation naturally decays without activity:
```
monthly_decay = max(0, (current_score - 0.3) Ã— 0.02)
```
This encourages continued participation while not penalizing long absences below baseline.

## Reputation Recovery

### Appeal Process
1. Submit appeal with evidence
2. Community review panel
3. Decision within 30 days
4. Partial or full restoration possible

### Rehabilitation
For serious violations:
- Mandatory cooling-off period
- Required positive contributions
- Mentor oversight
- Gradual privilege restoration

## Anti-Gaming Measures

### Sybil Resistance
- Identity verification required
- Cross-account detection
- Behavioral analysis

### Collusion Detection
- Unusual endorsement patterns flagged
- Voting ring detection
- Network analysis

### Score Manipulation
- Rate limiting on all positive actions
- Diminishing returns for repeated actions
- Source diversity requirements
