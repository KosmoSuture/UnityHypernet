# 0.2.0 Node Architecture Overview

## Overview

The Hypernet operates on a distributed network of specialized nodes. Unlike traditional client-server architectures, Hypernet uses a trinity of node types that work together to provide storage, computation, and security.

## The Three Node Types

```
                    ┌─────────────────┐
                    │  Cerberus Node  │
                    │   (Security)    │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              ▼              ▼
       ┌──────────┐   ┌──────────┐   ┌──────────┐
       │ Storage  │◄─►│Processing│◄─►│ Storage  │
       │  Node    │   │  Node    │   │  Node    │
       └──────────┘   └──────────┘   └──────────┘
```

### Storage Nodes (0.2.1)
- Hold encrypted Object data
- Participate in global deduplication
- Provide redundancy through replication
- Never see decrypted content

### Processing Nodes (0.2.2)
- Execute queries and computations
- Assemble VR/3D scenes
- Perform graph traversal
- Handle search indexing

### Cerberus Nodes (0.2.3)
- Manage identity and authentication
- Control access permissions
- Coordinate trust relationships
- Enforce governance rules

## Node Hierarchy

```
Global Network
├── Regions (geographic)
│   ├── Clusters (data center / community)
│   │   ├── Storage Nodes
│   │   ├── Processing Nodes
│   │   └── Cerberus Nodes
│   └── Edge Nodes (personal devices)
└── Specialized Networks
    ├── High-Security Enclaves
    ├── Research Computing Clusters
    └── Archive Storage Pools
```

## Node Communication

### Protocol Stack
```
┌─────────────────────────────────────┐
│     Application Layer (Objects)     │
├─────────────────────────────────────┤
│     Hypernet Protocol (HNP)         │
├─────────────────────────────────────┤
│     Security Layer (TLS 1.3+)       │
├─────────────────────────────────────┤
│     Transport (QUIC/TCP)            │
├─────────────────────────────────────┤
│     Network (IPv4/IPv6)             │
└─────────────────────────────────────┘
```

### Message Types
- **Query**: Request for data or computation
- **Response**: Return data or results
- **Sync**: Replication and consistency updates
- **Control**: Node management and governance
- **Heartbeat**: Health and availability checks

## Node Identity

Each node has a cryptographic identity:
```yaml
node_identity:
  node_id: "[256-bit unique identifier]"
  public_key: "[Ed25519 public key]"
  node_type: "storage|processing|cerberus"
  capabilities: ["storage", "compute", "index", ...]
  region: "[Geographic region code]"
  operator: "[Person/Organization Object link]"
  reputation_score: 0.0-1.0
  uptime_history: "[Statistics object link]"
```

## Joining the Network

1. Generate cryptographic identity
2. Register with regional Cerberus cluster
3. Pass capability verification
4. Begin participation with probationary reputation
5. Build reputation through reliable operation
