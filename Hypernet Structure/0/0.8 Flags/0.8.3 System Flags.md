# 0.8.3 — System Flags

System flags track technical and operational metadata about objects. They are primarily set by automated processes.

---

## 0.8.3.1 — ai-generated

```yaml
flag:
  address: "0.8.3.1"
  name: "ai-generated"
  description: "Content was generated by an AI instance"
  applied_by: ["ai_instances", "system_automated"]
  removed_by: ["governance_process"]
  mutually_exclusive: []
  triggers:
    - "Visual indicator showing AI origin"
    - "Links to generating AI instance (creator field)"
    - "May require human review before certain uses"
  metadata:
    generating_instance: "HA of the AI instance that created this"
    model: "Model identifier used (e.g., claude-opus-4-6)"
    tokens_used: "Number of tokens consumed in generation"
  visual: "sparkle icon, purple accent"
```

## 0.8.3.2 — auto-imported

```yaml
flag:
  address: "0.8.3.2"
  name: "auto-imported"
  description: "Content was automatically imported from an external source"
  applied_by: ["system_automated", "import_pipeline"]
  removed_by: ["content_owner"]
  mutually_exclusive: []
  triggers:
    - "Source tracking via source_type and source_id fields"
    - "May need manual review for accuracy"
  metadata:
    import_source: "Name of the import pipeline or external system"
    import_date: "ISO 8601 timestamp of import"
  visual: "download icon, blue accent"
```

## 0.8.3.3 — needs-migration

```yaml
flag:
  address: "0.8.3.3"
  name: "needs-migration"
  description: "Object needs to be migrated to a new format, schema, or location"
  applied_by: ["system_automated", "governance"]
  removed_by: ["migration_process"]
  mutually_exclusive: []
  triggers:
    - "Appears in migration queue"
    - "AI workers may auto-assign as maintenance task"
  metadata:
    migration_type: "schema_update | format_change | relocation | type_reclassification"
    target_version: "Version or format to migrate to"
  visual: "arrow-right icon, gray accent"
```
