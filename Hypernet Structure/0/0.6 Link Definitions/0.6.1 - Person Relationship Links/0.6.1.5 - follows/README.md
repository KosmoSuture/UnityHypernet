# 0.6.1.5 - follows

**Category:** Person Relationship (0.6.1)
**Directed:** Yes | **Symmetric:** No | **Transitive:** No
**Inverse:** followed_by
**Consent:** None required

## Description

Social following relationship. Person A follows Person B to receive updates on their activity. Does not require the target's consent.

## Properties

- **since**: When the follow started
- **notifications**: Whether to receive notifications
