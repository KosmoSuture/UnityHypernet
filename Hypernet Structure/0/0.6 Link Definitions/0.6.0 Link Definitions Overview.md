# 0.6.0 Link Definitions Overview

## Overview

Links are the third core component of the Hypernet, alongside the Library and Objects. They represent relationships between Objects, creating the interconnected web of knowledge that enables The Singularity's emergent intelligence.

## The Power of Links

In the Hypernet:
- **Objects** store information
- **The Library** addresses and organizes
- **Links** create meaning through relationships

Links transform isolated data into connected knowledge. A document becomes more valuable when linked to its author, sources, and related works. A person becomes part of a network when linked to colleagues, organizations, and contributions.

## Link Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                         Link Object                              │
├─────────────────────────────────────────────────────────────────┤
│  Source Object ─────────[Link Type]─────────► Target Object     │
│                              │                                   │
│                         Metadata                                 │
│                    (context, timestamps,                        │
│                     strength, evidence)                          │
└─────────────────────────────────────────────────────────────────┘
```

## Link Schema

```yaml
link:
  # ═══════════════════════════════════════════════════════════════
  # LINK IDENTITY
  # ═══════════════════════════════════════════════════════════════
  identity:
    link_id: "[UUID]"
    link_type: "[Type from 0.6.x]"

  # ═══════════════════════════════════════════════════════════════
  # ENDPOINTS
  # ═══════════════════════════════════════════════════════════════
  endpoints:
    source:
      address: "[Source Object address]"
      object_type: "person"

    target:
      address: "[Target Object address]"
      object_type: "organization"

  # ═══════════════════════════════════════════════════════════════
  # LINK PROPERTIES
  # ═══════════════════════════════════════════════════════════════
  properties:
    # Directionality
    directed: true  # true = one-way, false = bidirectional

    # Strength (for weighted relationships)
    weight: 0.8  # 0.0 to 1.0

    # Temporal validity
    temporal:
      valid_from: "2025-01-01T00:00:00Z"
      valid_until: null  # null = indefinite
      is_current: true

  # ═══════════════════════════════════════════════════════════════
  # METADATA
  # ═══════════════════════════════════════════════════════════════
  metadata:
    created:
      timestamp: "2025-01-30T12:00:00Z"
      by: "[Creator Mandala]"

    context: "Specific context for this relationship"

    evidence:
      - type: "document"
        reference: "[Document Object that evidences this link]"
      - type: "assertion"
        by: "[Person who asserted]"
        confidence: 0.9

    tags: ["professional", "verified"]

  # ═══════════════════════════════════════════════════════════════
  # VERIFICATION
  # ═══════════════════════════════════════════════════════════════
  verification:
    status: "unverified|self_attested|peer_verified|officially_verified"
    verifiers:
      - mandala: "[Verifier Mandala]"
        timestamp: "2025-01-30T14:00:00Z"
        method: "document_review"

    trust_score: 0.85
```

## Link Type Categories

### 0.6.1 Person Relationship Links
Connections between people:
- `knows` - Personal acquaintance
- `works_with` - Professional collaboration
- `related_to` - Family relationship
- `mentors` / `mentored_by` - Mentorship
- `reports_to` - Organizational hierarchy

### 0.6.2 Organizational Links
Connections involving organizations:
- `member_of` - Membership
- `employed_by` - Employment
- `founded` - Founding relationship
- `subsidiary_of` - Corporate structure
- `partners_with` - Partnership

### 0.6.3 Authorship and Creation Links
Content creation relationships:
- `created` / `created_by` - General creation
- `authored` / `authored_by` - Written content
- `contributed_to` - Partial contribution
- `edited` - Editorial relationship
- `owns` - Ownership

### 0.6.4 Reference and Citation Links
Knowledge connections:
- `cites` - Academic/formal citation
- `references` - General reference
- `quotes` - Direct quotation
- `derives_from` - Derived work
- `supersedes` - Replacement

### 0.6.5 Hierarchical Links
Structural relationships:
- `part_of` - Component relationship
- `contains` - Container relationship
- `parent_of` / `child_of` - Hierarchy
- `broader_than` / `narrower_than` - Taxonomy

### 0.6.6 Spatial Links
Location-based relationships:
- `located_at` - Physical location
- `near` - Proximity
- `within` - Containment
- `adjacent_to` - Adjacency

### 0.6.7 Temporal Links
Time-based relationships:
- `preceded_by` / `followed_by` - Sequence
- `during` - Temporal containment
- `caused` / `caused_by` - Causation
- `contemporary_with` - Simultaneity

### 0.6.8 Semantic Links
Meaning-based relationships:
- `similar_to` - Similarity
- `opposite_of` - Opposition
- `synonym_of` - Synonymy
- `instance_of` - Type-instance
- `example_of` - Exemplification

### 0.6.9 Task and Dependency Links
Work-related relationships:
- `assigned_to` - Task assignment
- `depends_on` - Dependency
- `blocks` / `blocked_by` - Blocking
- `subtask_of` - Task hierarchy
- `delivered` - Deliverable relationship

## Link Properties

### Directionality
- **Directed**: A → B (e.g., "A reports_to B")
- **Undirected**: A ↔ B (e.g., "A knows B")

### Symmetry
- **Symmetric**: If A→B then B→A (e.g., "married_to")
- **Asymmetric**: A→B doesn't imply B→A (e.g., "parent_of")

### Transitivity
- **Transitive**: A→B and B→C implies A→C (e.g., "ancestor_of")
- **Non-transitive**: No such implication (e.g., "knows")

### Cardinality
- **One-to-One**: Each source has one target
- **One-to-Many**: Source can have multiple targets
- **Many-to-Many**: Both can have multiple

## Link Verification

Links can be verified at multiple levels:

| Level | Description | Trust Impact |
|-------|-------------|--------------|
| Unverified | Just asserted | Low |
| Self-attested | Both parties confirm | Medium |
| Peer-verified | Third party confirms | High |
| Officially verified | Documentary evidence | Highest |

## Graph Operations

The link network enables:
- **Traversal**: Follow links to discover related objects
- **Pathfinding**: Find connections between any two objects
- **Clustering**: Identify groups of related objects
- **Inference**: Derive new links from existing patterns
- **Recommendation**: Suggest relevant content
