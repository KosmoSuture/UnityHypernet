# 0.6.2 Organizational Links

## Overview

Organizational links define relationships between people and organizations, and between organizations themselves. These links structure collaboration, employment, membership, and corporate relationships.

## Person-to-Organization Links

### member_of
General membership in an organization.

```yaml
link_type:
  name: "member_of"
  category: "membership"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["organization"]
  cardinality: "many_to_many"

  properties:
    role: "member role/title"
    department: "[Department reference]"
    since: "date"
    until: "date"  # null if current
    status: "active|inactive|emeritus|honorary"
    permissions: ["list of role permissions"]

  inverse: "has_member"

  verification:
    method: "organization_confirmation"
    requires: "organization admin approval"
```

### employed_by
Employment relationship.

```yaml
link_type:
  name: "employed_by"
  category: "employment"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["organization"]
  cardinality: "many_to_many"  # Person can have multiple employers

  properties:
    title: "job title"
    department: "[Department reference]"
    employment_type: "full_time|part_time|contract|intern"
    start_date: "date"
    end_date: "date"  # null if current
    compensation:  # Optional, encrypted
      type: "salary|hourly|contract"
      amount: "encrypted"
    supervisor: "[Person reference]"

  inverse: "employs"

  verification:
    method: "hr_system_verification"
    documents: ["offer_letter", "employment_contract"]
```

### founded
Founding relationship.

```yaml
link_type:
  name: "founded"
  category: "creation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["organization"]
  cardinality: "many_to_many"  # Multiple founders, person can found multiple orgs

  properties:
    founding_date: "date"
    role: "founder|co-founder"
    equity_percentage: "optional, encrypted"

  inverse: "founded_by"

  verification:
    method: "document_verification"
    documents: ["articles_of_incorporation", "founding_agreement"]
```

### contributes_to
Contribution relationship (especially for Hypernet).

```yaml
link_type:
  name: "contributes_to"
  category: "contribution"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["organization", "project"]
  cardinality: "many_to_many"

  properties:
    contribution_type: "code|content|design|management|support|financial"
    since: "date"
    total_hours: "number"
    credit_hours: "number"  # For profit sharing
    active: "boolean"

  inverse: "has_contributor"

  verification:
    method: "contribution_tracking_system"
```

## Organization-to-Organization Links

### subsidiary_of
Corporate ownership structure.

```yaml
link_type:
  name: "subsidiary_of"
  category: "corporate_structure"
  directed: true
  symmetric: false
  transitive: true  # If A subsidiary_of B, B subsidiary_of C, then A subsidiary_of C

  source_types: ["organization"]
  target_types: ["organization"]
  cardinality: "many_to_one"

  properties:
    ownership_percentage: "0-100"
    since: "date"
    relationship_type: "wholly_owned|majority|minority"

  inverse: "parent_of"

  verification:
    method: "legal_document_verification"
    documents: ["corporate_filings", "ownership_records"]
```

### partners_with
Partnership between organizations.

```yaml
link_type:
  name: "partners_with"
  category: "partnership"
  directed: false  # Symmetric
  symmetric: true
  transitive: false

  source_types: ["organization"]
  target_types: ["organization"]
  cardinality: "many_to_many"

  properties:
    partnership_type: "strategic|operational|research|distribution|technology"
    formalized: "boolean"
    agreement: "[Document reference]"
    since: "date"
    until: "date"
    scope: "description of partnership scope"

  verification:
    method: "mutual_confirmation"
    documents: ["partnership_agreement"]
```

### regulates / regulated_by
Regulatory relationship.

```yaml
link_type:
  name: "regulates"
  category: "regulatory"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["organization"]  # Regulatory body
  target_types: ["organization"]
  cardinality: "one_to_many"

  properties:
    jurisdiction: "[Location reference]"
    regulatory_domain: "financial|health|environmental|etc"
    license_numbers: ["list of licenses/registrations"]

  inverse: "regulated_by"

  verification:
    method: "regulatory_filing_verification"
```
