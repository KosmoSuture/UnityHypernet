## Overview

Task and dependency links structure work: what depends on what, who is assigned to what, what blocks what, and how deliverables flow through projects. These links power the Hypernet's task queue, project management, and the AI swarm's work coordination.

## Dependency Links

### depends_on / dependency_of
Task or object requires another to be completed first.

```yaml
link_type:
  name: "depends_on"
  address: "0.6.7.depends_on"
  category: "dependency"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["task", "document", "concept"]
  target_types: ["task", "document", "concept"]
  cardinality: "many_to_many"

  properties:
    dependency_type: "hard|soft|optional"
    # hard: cannot proceed without
    # soft: should complete first but can work in parallel
    # optional: helpful but not required
    satisfied: "boolean"        # Is the dependency met?
    satisfied_at: "timestamp"   # When it was met

  inverse: "dependency_of"

  verification:
    method: "system_automatic"
    auto_update: true           # System updates satisfied when dependency completes

  notes: |
    Critical for swarm task ordering. When the swarm picks its next task,
    it queries the depends_on graph to find tasks with all hard dependencies
    satisfied.
```

## Structure Links

### subtask_of / has_subtask
Task hierarchy.

```yaml
link_type:
  name: "subtask_of"
  address: "0.6.7.subtask_of"
  category: "task_hierarchy"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["task"]
  target_types: ["task"]
  cardinality: "many_to_one"   # Each subtask has one parent task

  properties:
    order: "integer"            # Sort order among sibling subtasks
    weight: "0.0-1.0"          # How much of parent this subtask represents
    can_parallel: "boolean"     # Can this run in parallel with siblings?

  inverse: "has_subtask"

  constraints:
    no_cycles: true
    max_depth: 10               # Prevent infinite nesting
```

## Task Dependency Graph Operations

| Operation | Description | Use Case |
|-----------|-------------|----------|
| **Critical Path** | Longest chain of hard depends_on links | Project planning |
| **Ready Queue** | Tasks with all depends_on satisfied | Swarm task selection |
| **Impact Analysis** | What breaks if this task fails | Risk assessment |
| **Bottleneck Detection** | Tasks that block the most others | Resource allocation |
| **Completion Forecast** | Estimate based on dependency chain lengths | Timeline projection |
