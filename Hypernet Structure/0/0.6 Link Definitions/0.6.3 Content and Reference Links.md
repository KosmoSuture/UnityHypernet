# 0.6.3 Content and Reference Links

## Overview

Content and reference links define how knowledge objects relate to each other and to their creators. These links form the citation network and provenance chain that enables truth verification in the Hypernet.

## Authorship Links

### authored_by / authored
Authorship of written content.

```yaml
link_type:
  name: "authored_by"
  category: "authorship"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    role: "primary_author|co_author|contributor|ghost_writer"
    contribution_description: "specific contribution"
    order: 1  # Author order (1 = first author)
    corresponding: "boolean"  # Is corresponding author

  inverse: "authored"

  verification:
    method: "document_metadata"
    cross_reference: "object.content.authorship"
```

link_type:

## Citation Links

### cites
Formal academic citation.

```yaml
link_type:
  name: "cites"
  category: "citation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document"]
  target_types: ["document"]
  cardinality: "many_to_many"

  properties:
    citation_type: "supports|contradicts|extends|reviews|methodology"
    location:
      section: "section in source document"
      page: "page number if applicable"
      paragraph: "paragraph reference"
    quote: "relevant quote from cited work"
    context: "how the citation is used"

  inverse: "cited_by"

  verification:
    method: "automated_verification"
    checks:
      - "target_document_exists"
      - "quote_matches_source"
      - "context_is_accurate"

  metrics:
    enables:
      - "citation_count"
      - "h_index"
      - "impact_factor"
```

### references
General reference (less formal than citation).

```yaml
link_type:
  name: "references"
  category: "reference"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document", "media", "task"]
  target_types: ["document", "media", "person", "organization", "location", "event"]
  cardinality: "many_to_many"

  properties:
    reference_type: "source|background|example|related"
    context: "how reference is used"

  inverse: "referenced_by"
```

### quotes
Direct quotation.

```yaml
link_type:
  name: "quotes"
  category: "quotation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document"]
  target_types: ["document", "person"]

  properties:
    quote_text: "exact quoted text"
    source_location:
      page: "number"
      timestamp: "for audio/video"
    attributed_to: "[Person reference]"  # If quoting a person
    date_of_quote: "date"

  inverse: "quoted_by"

  verification:
    method: "text_matching"
    accuracy_check: true
```

## Derivation Links

### derived_from
Derivative work relationship.

```yaml
link_type:
  name: "derived_from"
  category: "derivation"
  directed: true
  symmetric: false
  transitive: true  # If A derived_from B, B derived_from C, then A derived_from C

  source_types: ["document", "media", "concept"]
  target_types: ["document", "media", "concept"]

  properties:
    derivation_type: "translation|adaptation|remix|excerpt|analysis"
    transformation_description: "how source was transformed"
    license_compliance: "boolean"

  inverse: "source_of"

  verification:
    method: "license_verification"
```

### supersedes
Replacement relationship.

```yaml
link_type:
  name: "supersedes"
  category: "versioning"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["document", "concept"]
  target_types: ["document", "concept"]
  cardinality: "many_to_one"  # Many versions can supersede one

  properties:
    reason: "updated|corrected|expanded|replaced"
    effective_date: "date"
    backwards_compatible: "boolean"
    migration_guide: "[Document reference]"

  inverse: "superseded_by"

  verification:
    method: "self_attestation"
    requires: "same_owner_or_authority"
```

link_type:

## Evidence Links

### supports
Evidential support relationship.

```yaml
link_type:
  name: "supports"
  category: "evidence"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document", "media"]
  target_types: ["document", "concept"]  # Can support claims

  properties:
    claim_supported: "specific claim being supported"
    evidence_type: "empirical|logical|testimonial|documentary"
    strength: "0.0-1.0"

  inverse: "supported_by"

  verification:
    method: "expert_review"
    community_verification: true
```

### contradicts
Contradictory evidence.

```yaml
link_type:
  name: "contradicts"
  category: "evidence"
  directed: true
  symmetric: true  # If A contradicts B, B contradicts A
  transitive: false

  source_types: ["document"]
  target_types: ["document", "concept"]

  properties:
    contradiction_type: "factual|logical|methodological"
    explanation: "how they contradict"
    resolution: "if resolved, how"

  verification:
    method: "expert_review"
```
