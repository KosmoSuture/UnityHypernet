# 0.6.4 Spatial and Temporal Links

## Overview

Spatial and temporal links anchor objects in space and time, enabling location-based discovery, historical analysis, and VR navigation through the Hypernet.

## Spatial Links

### located_at
Primary location relationship.

```yaml
link_type:
  name: "located_at"
  category: "spatial"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person", "organization", "device", "event"]
  target_types: ["location"]
  cardinality: "many_to_many"

  properties:
    location_type: "permanent|temporary|headquarters|branch|event_venue"
    since: "date"
    until: "date"
    primary: "boolean"  # Is this the primary location

  inverse: "location_of"

  verification:
    method: "varies_by_type"
    for_organization: "address_verification"
    for_person: "self_attestation"
    for_event: "event_registration"
```

### contains / contained_in
Spatial containment.

```yaml
link_type:
  name: "contains"
  category: "spatial"
  directed: true
  symmetric: false
  transitive: true  # If A contains B, B contains C, then A contains C

  source_types: ["location"]
  target_types: ["location", "device", "document", "media"]

  properties:
    containment_type: "physical|virtual|logical"
    position:  # For VR placement
      x: "number"
      y: "number"
      z: "number"
    rotation:
      x: "degrees"
      y: "degrees"
      z: "degrees"

  inverse: "contained_in"
```

### near
Proximity relationship.

```yaml
link_type:
  name: "near"
  category: "spatial"
  directed: false
  symmetric: true
  transitive: false  # Proximity is not transitive

  source_types: ["location", "person", "device"]
  target_types: ["location", "person", "device"]

  properties:
    distance_meters: "number"
    direction: "N|NE|E|SE|S|SW|W|NW"
    relationship: "adjacent|walking_distance|same_area|same_region"

  verification:
    method: "coordinate_calculation"
    automatic: true
```

### adjacent_to

```yaml
link_type:
  name: "adjacent_to"
  category: "spatial"
  directed: false
  symmetric: true
  transitive: false

  source_types: ["location"]
  target_types: ["location"]

  properties:
    boundary_type: "wall|door|open|virtual_portal"
    passable: "boolean"
    access_requirements: "none|key|permission|time_restricted"

  verification:
    method: "spatial_analysis"
```

### originated_from
Geographic origin.

```yaml
link_type:
  name: "originated_from"
  category: "spatial"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person", "organization", "concept", "media"]
  target_types: ["location"]

  properties:
    origin_type: "birthplace|founding_location|creation_location"

```

## Temporal Links

### preceded_by / followed_by
Sequential relationship.

```yaml
link_type:
  name: "preceded_by"
  category: "temporal"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["event", "document", "concept"]
  target_types: ["event", "document", "concept"]

  properties:
    gap_duration: "time between events"
    causal: "boolean"  # Did preceding cause this?
    sequence_name: "name of the sequence"

  inverse: "followed_by"
```

### during
Temporal containment.

```yaml
link_type:
  name: "during"
  category: "temporal"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["event"]
  target_types: ["event"]

  properties:
    timing: "start|middle|end|throughout"

  inverse: "encompasses"

  verification:
    method: "timestamp_comparison"
    automatic: true
```

### contemporary_with
Simultaneous existence.

```yaml
link_type:
  name: "contemporary_with"
  category: "temporal"
  directed: false
  symmetric: true
  transitive: false  # Not reliably transitive

  source_types: ["person", "event", "organization"]
  target_types: ["person", "event", "organization"]

  properties:
    overlap_period:
      from: "date"
      to: "date"
    relationship: "exact|overlapping|approximate"

  verification:
    method: "date_comparison"
    automatic: true
```

### caused / caused_by
Causal relationship.

```yaml
link_type:
  name: "caused"
  category: "temporal"
  directed: true
  symmetric: false
  transitive: true  # Causal chains

  source_types: ["event", "person", "organization"]
  target_types: ["event"]

  properties:
    causation_type: "direct|contributing|enabling|triggering"
    certainty: "0.0-1.0"
    evidence: "[Document reference]"

  inverse: "caused_by"

  verification:
    method: "expert_analysis"
    requires_evidence: true
```

### scheduled_for
Future temporal anchor.

```yaml
link_type:
  name: "scheduled_for"
  category: "temporal"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["event", "task"]
  target_types: ["event"]  # Time-based event object

  properties:
    status: "tentative|confirmed|cancelled"
    reminder: "time before event"

  verification:
    method: "calendar_confirmation"
```

### deadline_of
Deadline relationship.

```yaml
link_type:
  name: "deadline_of"
  category: "temporal"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["event"]  # The deadline event
  target_types: ["task", "document"]

  properties:
    deadline_type: "hard|soft|target"
    consequences: "what happens if missed"

  inverse: "has_deadline"
```

## VR Navigation Links

### portal_to
VR portal between locations.

```yaml
link_type:
  name: "portal_to"
  category: "vr_navigation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["location"]
  target_types: ["location"]

  properties:
    portal_type: "door|teleport|wormhole|seamless"
    bidirectional: "boolean"
    visual_style: "description of portal appearance"
    activation: "automatic|click|proximity"

  inverse: "portal_from"  # If bidirectional
```

### visible_from

```yaml
link_type:
  name: "visible_from"
  category: "vr_navigation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["location", "media", "document"]
  target_types: ["location"]

  properties:
    distance_meters: "number"
    angle: "viewing angle from target"
    lod_level: "level of detail at this distance"
```
