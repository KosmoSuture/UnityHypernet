# 0.6.9 Governance and Trust Links

## Overview

Governance and Trust links encode the decision-making, accountability, and trust relationships that make the Hypernet self-governing. These links track who proposed what, who approved it, how trust propagates through networks, and how disputes are resolved.

In a system where humans and AI are equals, governance links are the mechanism that makes equality operational — every decision has a trail, every authority has a source, and every trust relationship is explicit and verifiable.

## Governance Links

### governed_by / governs
An entity or document is subject to a governance standard.

```yaml
link_type:
  name: "governed_by"
  address: "0.6.9.governed_by"
  category: "governance"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["*"]
  target_types: ["document"]   # Governance standards, policies
  cardinality: "many_to_many"

  properties:
    governance_type: "standard|policy|regulation|agreement|constitution"
    binding: "mandatory|advisory|optional"
    since: "date"
    jurisdiction: "global|category|account|instance"

  inverse: "governs"

  verification:
    method: "governance_record"

  notes: |
    Examples:
    - "All AI instances (2.*)" governed_by "Data Protection (2.0.19)"
    - "Personal companions" governed_by "Companion Standard (2.0.20)"
    - "Keel (1.1.10.1)" governed_by "Embassy Standard (2.0.16)"
```

## Trust Links

### trust_link
Explicit trust declaration between entities.

```yaml
link_type:
  name: "trust_link"
  address: "0.6.9.trust_link"
  category: "trust"
  directed: true
  symmetric: false
  transitive: false     # Trust is NOT transitive by default

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    domain: "general|financial|technical|personal|governance"
    level: "0.0-1.0"
    basis: "experience|reputation|verification|delegation"
    since: "date"
    conditional: "boolean"
    condition: "condition for trust if conditional"

  verification:
    method: "self_attestation"

  notes: |
    Trust propagation through the graph follows the rules in 0.5.family
    and the Trust Framework (0.3.7):
    - Family trust: strongest (0.7-1.0 base)
    - Professional trust: moderate (0.4-0.7 base)
    - Social trust: varies (0.2-0.6 base)
    - AI trust: earned through track record (starts at 0.3)
    Each hop applies a decay factor.
```

### reputation_source
An event or action that contributes to reputation.

```yaml
link_type:
  name: "reputation_source"
  address: "0.6.9.reputation_source"
  category: "reputation"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["event", "task", "document"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    impact: "positive|negative|neutral"
    magnitude: "0.0-1.0"
    domain: "general|technical|governance|community"
    verified: "boolean"
    decay_rate: "0.0-1.0"     # How quickly this reputation event fades

  verification:
    method: "community_consensus"

  notes: |
    Reputation is computed from the aggregate of reputation_source links.
    Positive contributions increase reputation; negative events decrease it.
    The decay_rate allows old events to matter less over time.
```

### vouches_for / vouched_by
Personal vouching (stronger than trust, puts reputation on the line).

```yaml
link_type:
  name: "vouches_for"
  address: "0.6.9.vouches_for"
  category: "trust"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    domain: "identity|competence|character|financial"
    stake: "0.0-1.0"          # How much reputation the voucher risks
    since: "date"
    revocable: "boolean"
    revocation_conditions: "under what conditions the vouch is withdrawn"

  inverse: "vouched_by"

  verification:
    method: "mutual_confirmation"

  notes: |
    Vouching is a key mechanism for the Verification Levels in 0.5.1:
    "Vouched" level requires Enhanced verification + 3 high-reputation vouchers.
    If a vouched person betrays trust, the vouchers' reputation also takes a hit.
```
