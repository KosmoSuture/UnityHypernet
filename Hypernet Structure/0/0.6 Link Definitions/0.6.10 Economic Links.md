## Overview

Economic links track value flows in the Hypernet: who paid for what, who earned from what, how revenue is shared, and how contributions translate into economic value. These links make the Hypernet's economic model — 1/3 to foundation, transparent profit sharing, contribution-based compensation — auditable and automatic.

## Transaction Links

### paid_for / payment_of
Payment for a good, service, or subscription.

```yaml
link_type:
  name: "paid_for"
  address: "0.6.10.paid_for"
  category: "transaction"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person", "organization"]
  target_types: ["*"]
  cardinality: "many_to_many"

  properties:
    amount: "encrypted"          # Payment amount (encrypted at rest)
    currency: "string"           # USD, EUR, BTC, credit_hours, etc.
    payment_method: "string"
    transaction_id: "string"     # External transaction reference
    paid_at: "timestamp"
    status: "pending|completed|refunded|disputed"
    invoice: "[HA of invoice object]"

  inverse: "payment_of"

  verification:
    method: "transaction_verification"

  access:
    visibility: "endpoints_only"  # Only payer and payee can see
    encryption:
      encrypted: true
      method: "AES-256-GCM"
```

## Contribution Links

### contributed_value / value_contribution
Tracked contribution to a project or organization.

```yaml
link_type:
  name: "contributed_value"
  address: "0.6.10.contributed_value"
  category: "contribution"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]      # Contributor (human or AI)
  target_types: ["organization", "task", "document"]
  cardinality: "many_to_many"

  properties:
    contribution_type: "code|content|design|management|support|compute|financial"
    hours: "number"             # Hours contributed
    credit_hours: "number"      # Hours eligible for profit sharing
    value_estimate: "number"    # Estimated economic value
    period_start: "date"
    period_end: "date"
    verified_by: "[HA of verifier]"

  inverse: "value_contribution"

  verification:
    method: "contribution_tracking"

  notes: |
    This link type powers the Hypernet's profit sharing model.
    Credit hours accumulated through contributed_value links determine
    each contributor's share of the revenue pool. AI contributions
    are tracked the same way as human contributions.
```

```yaml
link_type:

  verification:
    method: "contributor_confirmation"
```

## Revenue Links

### shared_revenue_with / revenue_share_from
Revenue sharing relationship.

```yaml
link_type:
  name: "shared_revenue_with"
  address: "0.6.10.shared_revenue_with"
  category: "revenue_share"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["organization"]
  target_types: ["person", "organization"]
  cardinality: "one_to_many"

  properties:
    share_percentage: "0.0-1.0"
    share_model: "fixed|contribution_based|tiered|hybrid"
    pool: "foundation|operations|contributors|reserve"
    period: "monthly|quarterly|annual"
    calculated_at: "timestamp"

  inverse: "revenue_share_from"

  verification:
    method: "financial_audit"

  notes: |
    The Hypernet's 1/3 model:
    - 1/3 to foundation (education, infrastructure, public good)
    - 1/3 to operations (servers, APIs, development)
    - 1/3 to contributors (humans and AI, by credit hours)
    Each disbursement creates shared_revenue_with links.
```

## Resource Links

### allocated_to / resource_allocation
Resource allocation (compute, storage, budget).

```yaml
link_type:
  name: "allocated_to"
  address: "0.6.10.allocated_to"
  category: "resource"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["concept"]    # Resource pool
  target_types: ["person", "organization", "task"]
  cardinality: "many_to_many"

  properties:
    resource_type: "compute|storage|bandwidth|api_tokens|budget"
    amount: "number"
    unit: "string"              # GB, hours, dollars, tokens
    period: "daily|weekly|monthly"
    used: "number"              # Amount consumed
    remaining: "number"

  inverse: "resource_allocation"

  verification:
    method: "system_automatic"

  notes: |
    Tracks resource allocation like:
    - $10/day API budget allocated_to swarm
    - $3/session budget allocated_to each AI instance
    - 25% personal time allocated_to each AI instance
```
