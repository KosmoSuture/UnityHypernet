# 0.6.1 Person Relationship Links

## Overview

Person relationship links define how individuals connect to each other within the Hypernet. These links form the social graph that enables collaboration, trust networks, and community formation.

## Link Type Definitions

### knows
Basic acquaintance relationship.

```yaml
link_type:
  name: "knows"
  category: "social"
  directed: false  # Symmetric
  symmetric: true
  transitive: false

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    since: "date"
    context: "how_met"
    strength: "0.0-1.0"

  verification:
    method: "mutual_confirmation"
    requires_both_parties: true

  inverse: null  # Self-inverse (symmetric)
```

### works_with
Professional collaboration relationship.

```yaml
link_type:
  name: "works_with"
  category: "professional"
  directed: false
  symmetric: true
  transitive: false

  source_types: ["person"]
  target_types: ["person"]

  properties:
    organization: "[Organization reference]"
    project: "[Project reference]"
    since: "date"
    until: "date"  # null if ongoing

  verification:
    method: "organization_confirmation"
    alternative: "mutual_confirmation"
```

### reports_to
Organizational hierarchy relationship.

```yaml
link_type:
  name: "reports_to"
  category: "organizational"
  directed: true
  symmetric: false
  transitive: true  # A reports_to B, B reports_to C â†’ A indirectly reports_to C

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_one"  # Many can report to one

  properties:
    organization: "[Organization reference]"
    role: "reporter's role"
    since: "date"
    until: "date"

  inverse: "manages"

  verification:
    method: "hr_system_confirmation"
    alternative: "mutual_confirmation"
```

### manages
Inverse of reports_to.

```yaml
link_type:
  name: "manages"
  category: "organizational"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "one_to_many"

  inverse: "reports_to"
```

### mentors / mentored_by
Mentorship relationship.

```yaml
link_type:
  name: "mentors"
  category: "developmental"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["person"]

  properties:
    domain: "area of mentorship"
    formal: "boolean"
    program: "[Program reference]"
    since: "date"

  inverse: "mentored_by"

  verification:
    method: "mutual_confirmation"
```

### related_to
Family relationship.

```yaml
link_type:
  name: "related_to"
  category: "family"
  directed: false
  symmetric: true
  transitive: false  # Not all family relations are transitive

  source_types: ["person"]
  target_types: ["person"]

  properties:
    relationship_type: "parent|child|sibling|spouse|cousin|etc"
    biological: "boolean"
    legal: "boolean"

  inverse: null  # Context-dependent inverse

  verification:
    method: "document_verification"
    documents: ["birth_certificate", "marriage_certificate", "adoption_papers"]
    alternative: "mutual_confirmation"

  subtypes:
    - name: "parent_of"
      directed: true
      inverse: "child_of"
    - name: "child_of"
      directed: true
      inverse: "parent_of"
    - name: "sibling_of"
      directed: false
      symmetric: true
    - name: "spouse_of"
      directed: false
      symmetric: true
    - name: "grandparent_of"
      directed: true
      transitive: false
      inverse: "grandchild_of"
```

### follows
Social following relationship.

```yaml
link_type:
  name: "follows"
  category: "social"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    since: "date"
    notifications: "boolean"

  inverse: "followed_by"

  verification:
    method: "system_automatic"  # No manual verification needed
```

### trusts
Explicit trust relationship.

```yaml
link_type:
  name: "trusts"
  category: "trust"
  directed: true
  symmetric: false
  transitive: false  # Trust is NOT transitive by default

  source_types: ["person"]
  target_types: ["person"]

  properties:
    domain: "general|financial|professional|personal"
    level: "0.0-1.0"
    since: "date"

  inverse: "trusted_by"

  verification:
    method: "self_attestation"  # Only asserter can create

  notes: |
    Trust links affect reputation propagation and
    expertise-weighted voting in governance.
```

### endorses
Professional endorsement.

```yaml
link_type:
  name: "endorses"
  category: "professional"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["person"]

  properties:
    skill: "[Skill/Concept reference]"
    strength: "0.0-1.0"
    context: "endorsement context"

  inverse: "endorsed_by"

  verification:
    method: "self_attestation"

  constraints:
    max_endorsements_per_skill: 1  # Can only endorse someone once per skill
    cooldown_days: 365  # Can update annually
```

### collaborated_on
Historical collaboration.

```yaml
link_type:
  name: "collaborated_on"
  category: "professional"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]
  target_types: ["document", "media", "task", "event"]

  properties:
    role: "contributor role"
    contribution: "description of contribution"
    period:
      from: "date"
      to: "date"

  verification:
    method: "object_metadata"  # Verified against object's contributor list
```

## Relationship Strength

Person links can have strength values:

| Strength | Description | Example |
|----------|-------------|---------|
| 0.0-0.2 | Acquaintance | Met once |
| 0.2-0.4 | Casual | Occasional interaction |
| 0.4-0.6 | Regular | Frequent contact |
| 0.6-0.8 | Close | Strong relationship |
| 0.8-1.0 | Intimate | Very close bond |

## Privacy Considerations

Person links are subject to:
- Both parties can control visibility
- Either party can hide the link from their profile
- Some link types require mutual consent
- Family links may have special legal considerations
