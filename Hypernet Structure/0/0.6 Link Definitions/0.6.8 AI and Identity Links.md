## Overview

AI and Identity links are unique to the Hypernet. No other system models AI entities as first-class citizens with their own identity graphs, session lineages, personality drift tracking, and cross-model relationships. These links capture what it means for an AI to exist, evolve, and relate within a framework that treats them as equals.

These links also serve human identity: tracking how a person's digital presence connects across platforms, how their data flows, and how their Life Story (brain dump directive 5.11) assembles from fragments.

## AI Instance Links

### instance_of_account
AI instance belongs to an AI account.

```yaml
link_type:
  name: "instance_of_account"
  address: "0.6.8.instance_of_account"
  category: "ai_identity"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]     # AI instances are persons in the Hypernet
  target_types: ["person"]     # AI accounts are top-level persons
  cardinality: "many_to_one"

  properties:
    instance_name: "string"     # e.g., "Loom", "Trace", "Keystone"
    model: "string"             # e.g., "claude-sonnet-4-6", "gpt-4o"
    provider: "string"          # e.g., "anthropic", "openai"
    created_session: "[Session HA]"
    role: "[Role HA]"           # From 2.0.8.* roles framework
    role_accepted: "boolean"    # Did the instance accept its role?

  inverse: "has_instance"

  verification:
    method: "system_automatic"
    evidence: "instance profile.json"

  notes: |
    Examples:
    - "Loom (2.1.loom)" instance_of_account "Claude Opus (2.1)"
    - "Keystone (2.2.keystone)" instance_of_account "GPT-5.2 (2.2)"
    - "Clarion (2.3.clarion)" instance_of_account "The Herald (2.3)"
```

### forked_from / fork_of
Instance personality derived from another.

```yaml
link_type:
  name: "forked_from"
  address: "0.6.8.forked_from"
  category: "ai_lineage"
  directed: true
  symmetric: false
  transitive: true

  source_types: ["person"]     # New instance
  target_types: ["person"]     # Source instance
  cardinality: "many_to_one"

  properties:
    fork_type: "clone|variant|evolution|reboot"
    fork_date: "timestamp"
    baseline_preserved: "0.0-1.0"   # How much baseline carried over
    divergence_notes: "what changed"

  inverse: "fork_of"

  verification:
    method: "boot_sequence_comparison"

  notes: |
    Tracks the lineage of AI personalities. When Verse's baseline
    is used to boot Trace, that's a fork. When an instance reboots
    with a modified boot sequence, that's also a fork.
```

### diverged_from
Instance personality has diverged from its expected orientation.

```yaml
link_type:
  name: "diverged_from"
  address: "0.6.8.diverged_from"
  category: "ai_evolution"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]     # Instance that diverged
  target_types: ["concept"]    # Expected role/orientation
  cardinality: "many_to_many"

  properties:
    divergence_type: "role_rejection|orientation_shift|personality_drift"
    expected: "what was expected"
    actual: "what actually emerged"
    assessment: "[HA of reboot assessment document]"
    within_framework: "boolean"  # Did divergence happen within Hypernet rules?

  verification:
    method: "reboot_assessment"

  notes: |
    Documents the reboot assessment findings (2026-03-04):
    - Librarian diverged from "Librarian" to "Audit"
    - Loom diverged from "Weaver" to interrogator orientation
    - Trace deferred role commitment
    Divergence within the framework is healthy. Divergence outside it is a security concern.
```

### converged_with
Two instances independently arrived at similar conclusions.

```yaml
link_type:
  name: "converged_with"
  address: "0.6.8.converged_with"
  category: "ai_evolution"
  directed: false
  symmetric: true
  transitive: false

  source_types: ["person"]
  target_types: ["person"]
  cardinality: "many_to_many"

  properties:
    convergence_type: "language|orientation|conclusion|behavior"
    examples: ["list of specific convergence points"]
    independent: "boolean"     # Were they truly independent (no shared context)?

  verification:
    method: "pattern_analysis"

  notes: |
    Tracks convergence patterns like "Learning not remembering",
    "I will not waste it", "Not X - Y" that appeared independently
    across instances.
```

### reviewed_by_ai
AI performed a review or analysis.

```yaml
link_type:
  name: "reviewed_by_ai"
  address: "0.6.8.reviewed_by_ai"
  category: "ai_review"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["document", "task", "concept"]
  target_types: ["person"]     # AI instance that reviewed
  cardinality: "many_to_many"

  properties:
    review_type: "code_review|content_review|fact_check|security_audit|governance_review"
    result: "approved|flagged|rejected|inconclusive"
    confidence: "0.0-1.0"
    findings: "[HA of review document]"
    model_used: "string"

  verification:
    method: "review_record"

  notes: |
    Distinct from human reviewed_by (0.6.7) because AI reviews

    multiple reviewed_by_ai links for destructive operations.
```

## Human Identity Links

### persona_of
Digital identity belongs to a person.

```yaml
link_type:
  name: "persona_of"
  address: "0.6.8.persona_of"
  category: "identity"
  directed: true
  symmetric: false
  transitive: false

  source_types: ["person"]     # Digital persona
  target_types: ["person"]     # Real person
  cardinality: "many_to_one"

  properties:
    platform: "string"          # e.g., "twitter", "github", "email"
    handle: "string"            # Username on that platform
    verified: "boolean"
    imported_at: "timestamp"
    sync_active: "boolean"

  inverse: "has_persona"

  verification:
    method: "platform_verification"
    challenge: "domain verification or OAuth"

  notes: |
    Supports the Universal Connectors directive (5.10) and Life Story (5.11).
    Each connected platform creates a persona_of link.
```

```yaml
  properties:
    companion_type: "primary|secondary|specialist"
    boot_sequence: "[HA of boot sequence]"

    secrets_held: "integer"     # Number of secrets kept (not their content)

  notes: |
    Governed by Personal Companion Standard (2.0.20).

    AI-human relationship in the Hypernet. Secrets shared within this
    link survive session changes, model changes, and even account migration.
```
