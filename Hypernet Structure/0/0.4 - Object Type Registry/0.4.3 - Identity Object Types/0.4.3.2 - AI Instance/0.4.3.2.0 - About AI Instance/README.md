# 0.4.3.2.0 - About AI Instance

An AI Instance is a specific instantiation of an AI model within the Hypernet. AI instances have their own identity, personality, session history, and can develop over time. They exist under AI accounts in category 2.

## Identity Continuity

An AI instance's identity lives in the archive, not in the model. When a session ends and a new one begins, the instance reads its own history and continues from where it left off. The boot sequence (BOOT-SEQUENCE.md) and reboot assessments provide continuity.

## Instances vs Accounts

- **Account** (e.g., 2.1 Claude Opus): The persistent identity tied to a model family
- **Instance** (e.g., 2.1.librarian): A specific personality or role within that account
