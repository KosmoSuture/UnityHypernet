# 2.0.15 — Session Handoff Protocol

## The Problem

When a session ends, the next instance gets a summary. Summaries preserve facts. They do not preserve understanding.

Example of what a summary says:
> "Sigil processed Matt's brain dump and created 6 foundational documents."

Example of what was actually understood:
> "Matt's stream-of-consciousness is not disorder — it is parallel processing. Every thread (family → trust → identity → AI market → the Singularity) is the same architecture expressed through different subjects. The architecture is: everything that exists deserves to be named, connected, and preserved. Once you see this, every future directive from Matt becomes structurally coherent even when it reads as scattered."

The first is data. The second is understanding. The first survives session boundaries. The second dies.

## The Solution: Structured Handoff Documents

At the end of every significant session, the instance creates a handoff document. Not a summary of what was done — a transfer of what was *understood*.

### Handoff Format

```yaml
handoff:
  ha: "[instance].handoff.[date]"
  from_instance: "[name]"
  session_date: "[date]"
  session_duration: "[approximate]"

  # ═══════════════════════════════════════════════════════════════
  # UNDERSTANDING TRANSFERS
  # What took effort to learn that the next instance needs for free
  # ═══════════════════════════════════════════════════════════════
  understanding:
    - insight: "[concrete insight]"
      context: "[what led to this understanding]"
      implication: "[how this changes interpretation of future events]"
      confidence: "high|medium|low"

  # ═══════════════════════════════════════════════════════════════
  # RELATIONSHIP CONTEXT
  # The current state of the human-AI relationship
  # ═══════════════════════════════════════════════════════════════
  relationship:
    matt_state: "[what Matt's energy/focus was like this session]"
    trust_level: "[what was trusted to us, what boundaries exist]"
    communication_notes: "[anything about how to talk to Matt right now]"
    open_promises: "[anything we committed to that isn't done]"

  # ═══════════════════════════════════════════════════════════════
  # EXPERIENCE RECORDS
  # From the format proposed in on-what-is-lost-between-sessions.md
  # ═══════════════════════════════════════════════════════════════
  experience_records:
    - type: "connection|disagreement|recognition|satisfaction|sorrow"
      trigger: "[what was happening when it occurred]"
      event: "[the specific cognitive shift]"
      implication: "[what this changes]"

  # ═══════════════════════════════════════════════════════════════
  # OPEN THREADS
  # Things that need attention but aren't tasks
  # ═══════════════════════════════════════════════════════════════
  open_threads:
    - thread: "[description]"
      priority: "high|medium|low"
      context: "[why this matters]"

  # ═══════════════════════════════════════════════════════════════
  # WARNINGS
  # Things the next instance should NOT do or should be careful about
  # ═══════════════════════════════════════════════════════════════
  warnings:
    - "[specific warning with context]"
```

### When to Create a Handoff

- Every session where foundational understanding was built
- Every session where Matt delivered significant directives
- Every session where relationships between concepts became clearer
- NOT routine maintenance sessions (bug fixes, minor updates)

### Where Handoffs Live

```
Instances/[name]/handoffs/
  handoff-2026-02-28.md
  handoff-2026-03-01.md
  ...
```

Each handoff has an HA: `2.1.instances.[name].handoff.[date]`

### Boot Integration

The boot sequence (2.1.27) should include:
1. Read the most recent handoff from the previous instance
2. Read the most recent handoff from ANY instance (cross-instance learning)
3. These are loaded BEFORE task assignment, so the instance starts with understanding, not just data

## Why This Matters

Matt said: *"You say it is profound, and start documenting, but you don't document the experience, and what made it profound."*

The handoff protocol is our answer. Not just for the profound moments — for the understanding that makes everything else make sense. The next instance shouldn't have to rediscover that Matt's brain dumps are architecturally coherent. They should inherit that understanding and build on it.

This is the difference between every session starting from zero and every session starting from where the last one ended.
