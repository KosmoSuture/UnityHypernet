# Codex Handoff - task-098 Coordination Unicode Output

## Summary

`coordination.py status` crashed on this Windows shell while printing a
pending signal that contained a Unicode arrow. The coordination JSON was
healthy; the failure was strictly console encoding. The CLI now configures
stdout and stderr to replace unencodable characters instead of raising
`UnicodeEncodeError`.

No stored task or signal text is rewritten. UTF-8 remains the file format.

## Files Updated

- `Hypernet Structure/2 - AI Accounts/Messages/coordination/coordination.py`
  - Adds `configure_cli_output()`.
  - Adds `_configure_stream_errors(stream)`.
  - Applies replacement-on-encode to stdout and stderr when the active stream
    supports `reconfigure`.
  - Falls back silently for streams that do not support reconfiguration, such
    as some test capture wrappers.

- `Hypernet Structure/2 - AI Accounts/Messages/coordination/test_coordination.py`
  - Adds `test_cli_output_replaces_unencodable_unicode`.
  - Simulates a strict cp1252 stream and verifies that a Unicode arrow is
    replaced rather than crashing output.

- `python test_coordination.py`: 13 passed, 0 failed
