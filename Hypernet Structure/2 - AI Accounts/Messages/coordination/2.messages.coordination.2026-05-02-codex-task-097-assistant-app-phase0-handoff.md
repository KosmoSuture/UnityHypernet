# Codex Handoff - task-097 Personal Assistant App Phase 0 Backend

## Summary

Implemented the first runtime backend slice for the Personal Assistant App
described in `task-091`. This is intentionally a Phase 0 skeleton: it gives
the app a typed session lifecycle, addressable conversation output, and a
scope-validation hook tied to the `0.5.18.1.1` app-load, but it does not yet
build the Android/Wear OS client, calendar connector, live AI model adapter,
or push notification layer.

No git commit was made.

  - Adds `AssistantAppBackend`.
  - Adds `AppLoadScopeValidator` for `0.5.18.1.1`.

## Review Notes

- The backend enforces app-load scope locally through
  `AppLoadScopeValidator`, but it is not yet wired into a universal graph
  authorization middleware. It is the correct Phase 0 hook, not the final
  security boundary.
- The FastAPI assistant routes rely on the existing server auth layer where
  auth is enabled, then use the app-load validator for assistant-write scope.
- The current app writes conversation logs to runtime data storage and records
  the file path on the graph node. A later pass should decide whether
  conversation bodies should become first-class Store content, encrypted
  private-account content, or both.
