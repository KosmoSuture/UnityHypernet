# Codex Handoff - task-099 Assistant App Model Adapter

## Summary

Added a Phase 0.5 model-adapter seam to the Personal Assistant App backend.
The app can now optionally generate a deterministic local assistant response
when a client posts a user turn with `auto_respond: true`. This gives the
Android/Wear OS client work a stable backend contract without requiring live
OpenAI, Claude, local model, or private-secret integration yet.

No git commit was made.

## Files Updated

- `Hypernet Structure/0/0.1 - Hypernet Core/hypernet/assistant_app.py`
  - Adds `AssistantResponse`.
  - Adds `AssistantModelAdapter` protocol.
  - Adds `DeterministicAssistantAdapter`.
  - Adds optional `model_adapter` injection to `AssistantAppBackend`.
  - Extends `add_turn(..., auto_respond=True)` to call the adapter when
    `assistant_text` is not provided.
  - Records response provenance metadata: `response_source`, `provider`,
    `model`, adapter name, generated flag, user-turn count, and input preview.

## Review Notes

- This is not a live model adapter. It is a stable interface and deterministic
  local implementation for development, tests, and early client integration.
- Provided assistant text still wins. `auto_respond` is only used when the
  caller does not provide `assistant_text`.
- The live provider adapter should load secrets only through the private
  account/secrets path and should record provider/model metadata on every
  generated turn.
