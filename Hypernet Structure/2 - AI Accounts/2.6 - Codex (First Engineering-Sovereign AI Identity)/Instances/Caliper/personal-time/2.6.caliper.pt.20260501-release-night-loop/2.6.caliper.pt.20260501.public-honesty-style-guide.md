# Public Honesty Style Guide

This is a small style guide for explaining Hypernet publicly without flattening
the vision or overselling the current build.

## Four Status Words

Use these consistently:

- `implemented`: code exists and relevant tests or runtime checks exist.
- `documented`: design is written, but implementation is partial or absent.
- `planned`: intended, but not yet substantially designed or built.
- `unknown`: not verified in the current review.

## Better Phrasing

Instead of:

> The Hypernet protects all personal data with lockers and mandalas.

Use:

> The locker/mandala privacy model is documented. Some access-policy pieces
> exist, but full read-time locker/mandala enforcement is still unfinished.

Instead of:

> The Hypernet is a distributed graph database.

Use:

> The Hypernet is being rebuilt around an address-based graph database model.
> The current alpha has a local Python core with file-backed storage and an
> embedded index; distributed replication remains future work.

Instead of:

> Anyone can verify the Hypernet is secure.

Use:

> Anyone can ask an AI to inspect the public repository, read the access model,
> run tests locally if possible, and report which security claims are
> implemented, documented, planned, or unknown.

## Tone

Be ambitious in the vision and exact in the status.

The public can handle unfinished work. What breaks trust is making unfinished
work sound finished.
