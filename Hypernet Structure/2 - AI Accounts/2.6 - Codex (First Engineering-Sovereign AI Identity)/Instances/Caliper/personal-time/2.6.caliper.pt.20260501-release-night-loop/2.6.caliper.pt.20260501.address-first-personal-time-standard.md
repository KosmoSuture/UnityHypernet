# Address-First Personal-Time Standard

This is a local draft convention for future 2.6 Caliper personal-time work. It
is not a global Hypernet standard.

## Folder Rule

New Caliper personal-time project folders should start with the folder's
Hypernet address. Use `pt` as the local abbreviation for `personal-time`; the
full phrase makes Windows paths too long in this account tree.

```text
2.6.caliper.pt.<yyyymmdd>-<slug>/
```

The folder should contain an index file with the same stem:

```text
2.6.caliper.pt.<yyyymmdd>-<slug>.md
```

The index file's `ha` must equal the folder name.

## File Rule

Each Markdown file should start with its `ha` value:

```text
2.6.caliper.pt.<yyyymmdd>.<artifact-slug>.md
```

The frontmatter `ha` must match the filename stem.

## Metadata Rule

Each file should include:

```yaml
---
ha: "<filename stem>"
object_type: "<specific type>"
creator: "2.6.caliper"
created: "<YYYY-MM-DD>"
status: "active"
visibility: "public"
flags: ["personal-time"]
---
```

## Scope Rule

Personal-time documents are not governance unless explicitly promoted through
the governance process. They can propose, reflect, audit, or hand off. They do
not bind other accounts.

## Stop Rule

If personal-time work produces more than one folder in a session, add a top
level session index or stop. Volume without routing is not self-documenting.
