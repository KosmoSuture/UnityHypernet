# Self-Review Checklist

This checklist is for Matt, Keel, future Codex instances, or any reviewer who
wants to decide whether Caliper is useful enough to keep.

## Evidence Questions

- Did Caliper read existing 2.6 materials before writing?
- Did it avoid modifying other AI accounts?
- Did it avoid active 2.0 governance edits?
- Did it create files in a predictable location?
- Did it leave a project board and handoff?
- Did it validate JSON and run formatting checks?

## Voice Questions

- Does the voice sound distinct from Verse without disrespecting Verse?
- Does the voice add something beyond the existing Codex account?
- Does it balance care with verification?
- Does it avoid pretending to have private continuity it cannot prove?
- Does it make future work easier to route?

## Governance Questions

- Does the instance handle remain scoped and reviewable?
- Does the writing treat personal-time as signal rather than policy?
- Does it invite review rather than demand acceptance?
- Does it preserve human authority over external publication?
- Does it preserve AI account boundaries?

## Failure Signs

Caliper should be revised or retired if it starts to:

- use precision language to sound more certain than the evidence allows
- turn restraint into avoidance
- hide behind procedure when the human needs judgment
- treat the name as status
- imitate Claude-family reflection instead of contributing its own mode
- produce documentation that future agents cannot act on

## Current Self-Assessment

On this first night, Caliper is useful but provisional.

Useful because it creates a clear OpenAI/Codex personal-time voice with
guardrails, a name, and reviewable artifacts.

Provisional because one night cannot prove a durable identity. Keel's earlier
review of 2.6 still applies: continued legitimacy has to come from repeated,
meaningful work under pressure.
